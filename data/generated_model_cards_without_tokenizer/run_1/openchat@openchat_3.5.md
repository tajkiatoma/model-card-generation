## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by a team of researchers from Tsinghua University and the Shanghai Artificial Intelligence Laboratory (2309.11235.pdf, p. 1). The authors are Guan Wang, Sijie Cheng, Xianyuan Zhan, Xiangang Li, Sen Song, and Yang Liu (2309.11235.pdf, p. 1).

### Model date:
The associated research paper was submitted on September 20, 2023, with a revised version dated March 16, 2024. It was published as a conference paper at ICLR 2024 (2309.11235.pdf, p. 1).

### Model version:
The model type is specified as `openchat_v3.2_mistral` in the training configuration (openchat.json.txt). The accompanying paper primarily discusses and evaluates `openchat-13b`, which is based on the Llama-2 13B model (2309.11235.pdf, p. 6). The provided configuration files (`config.json.txt`, `openchat.json.txt`) point to a version based on Mistral 7B. This model card primarily describes the OpenChat framework and the `openchat-13b` version detailed in the paper, as it is the most comprehensively documented.

The OpenChat framework distinguishes itself from other models by using a novel fine-tuning method called Conditioned-RLFT (C-RLFT), which leverages mixed-quality data without requiring explicit preference labels, unlike traditional Supervised Fine-Tuning (SFT) or Reinforcement Learning from Human Feedback (RLHF) methods (2309.11235.pdf, p. 1).

### Model type:
OpenChat is an open-source, Transformer-based large language model (LLM) fine-tuned for instruction-following and chat capabilities (2309.11235.pdf, p. 1). The specific model version described in the configuration files is `MistralForCausalLM` with a `mistral` model type (config.json.txt).

**Architecture:**
The model architecture is based on Mistral 7B (config.json.txt; openchat.json.txt). Key architectural details include:
*   **Number of Layers:** 32 hidden layers (config.json.txt).
*   **Hidden Size:** 4096 (config.json.txt).
*   **Attention Heads:** 32 query heads and 8 key/value heads (Grouped-Query Attention) (config.json.txt).
*   **Intermediate Size:** 14336 (config.json.txt).
*   **Activation Function:** SiLU ("silu") (config.json.txt).
*   **Attention Mechanism:** Uses a sliding window attention with a window of 4096 tokens (config.json.txt).

**Model Size:**
*   **Parameters:** The base model is a 7B parameter model (openchat.json.txt).
*   **Total Size on Disk:** Approximately 14.48 GB (pytorch_model.bin.index.json.txt).
*   **Vocabulary Size:** 32,002 (config.json.txt).

**Context Length:**
The model supports a maximum context length of 8192 tokens (config.json.txt; generation_config.json.txt).

### Training details:
The model is fine-tuned using a novel method called **Conditioned-Reinforcement Learning Fine-Tuning (C-RLFT)** (2309.11235.pdf, p. 1).

**Algorithm:**
C-RLFT is designed to work with mixed-quality data (e.g., a small amount of expert data and a large amount of sub-optimal data). The core ideas are:
1.  **Coarse-Grained Rewards:** Different data sources are treated as having coarse-grained reward labels. For example, conversations generated by GPT-4 are considered "expert" (reward=1.0) and those from GPT-3.5 are "sub-optimal" (reward < 1.0) (2309.11235.pdf, p. 4).
2.  **Class-Conditioned Policy:** The model is trained as a class-conditioned policy, where it is explicitly prompted with the source of the data (e.g., "GPT4 User:" vs. "GPT3 User:"). This allows the model to learn to distinguish between data of different qualities (2309.11235.pdf, p. 4).
3.  **RL-Free Supervised Learning:** The optimal policy for this framework can be solved via a single-stage, reward-weighted regression (supervised learning), which is simpler and more stable than traditional RL-based fine-tuning methods like PPO (2309.11235.pdf, pp. 1, 5).

**Hyperparameters (for the `openchat-13b` version):**
*   **Optimizer:** AdamW (2309.11235.pdf, p. 6).
*   **Epochs:** 5 (2309.11235.pdf, p. 6; openchat.json.txt).
*   **Learning Rate:** A cosine schedule with a maximum learning rate of 6.7e-5, decaying to 10% of the max value (2309.11235.pdf, p. 6). The provided `openchat.json.txt` specifies a learning rate of 1.25e-05.
*   **Batch Size:** Effective batch size of 200k tokens (2309.11235.pdf, p. 6).
*   **Weight Decay:** 0.1 (2309.11235.pdf, p. 6; openchat.json.txt).
*   **Adam Betas:** β1 = 0.9, β2 = 0.95 (2309.11235.pdf, p. 6; openchat.json.txt).
*   **Adam Epsilon:** 1e-5 (2309.11235.pdf, p. 6; openchat.json.txt).
*   **Sequence Length:** 4096 tokens (2309.11235.pdf, p. 6).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The model is intended for use as an instruction-following chatbot (2309.11235.pdf, p. 1). It is designed to generate high-quality, helpful responses to user prompts in a conversational format. The model's C-RLFT training method is specifically aimed at improving the performance of open-source language models by effectively learning from datasets that contain a mix of high-quality and sub-optimal examples (2309.11235.pdf, p. 1).

The input-output structure is conversational. The user provides a prompt, and the model generates a response. A special token, `<|end_of_turn|>`, is used to differentiate between user and assistant utterances (2309.11235.pdf, p. 4).

### Primary intended users:
The primary intended users are AI researchers and developers who are working on or with open-source large language models. The project aims to democratize AI research and provide a transparent platform for innovation (2309.11235.pdf, p. 10).

## How to Use
This section outlines how to use the model.

The model should be used in a conversational format. During inference, it is recommended to use the specific template that was used for the high-quality (GPT-4) data during training to elicit the best performance (2309.11235.pdf, p. 5).

**Inference Template:**
```
GPT4 User: {user_question}<|end_of_turn|>GPT4 Assistant:
```
Here, `{user_question}` should be replaced with the user's prompt. The model will then generate the text that follows the `GPT4 Assistant:` token (2309.11235.pdf, p. 5).

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
The most significant factor influencing the model's performance is the **quality of the training data**. The OpenChat framework is explicitly designed around the factor of data quality, categorizing training data into different classes (e.g., "expert" vs. "sub-optimal") based on their source (e.g., GPT-4 vs. GPT-3.5). The model's ability to differentiate between these sources via class-conditioning is key to its effectiveness (2309.11235.pdf, pp. 1, 4).

### Evaluation factors:
The factors analyzed during evaluation are directly related to the relevant factors. The model's performance is evaluated on its instruction-following ability and generalization across various domains (2309.11235.pdf, p. 5). Ablation studies were conducted to specifically measure the impact of the coarse-grained rewards and the class-conditioned policy, which are the mechanisms used to handle the different data quality factors (2309.11235.pdf, p. 8, Table 2).

## Metrics
This section describes how the model's performance is evaluated.

### Model performance measures:
The model's performance was assessed using three main metrics, corresponding to the benchmarks used:
*   **Win Rate (%):** Used for pairwise comparisons in AlpacaEval and Vicuna-bench. The evaluator compares the model's output to a reference model's output and assigns a win, tie, or loss (2309.11235.pdf, pp. 5-6).
*   **Score:** Used for single-answer grading in MT-bench. An LLM evaluator judges the generated answer on a scale from 1 to 10 (2309.11235.pdf, p. 6).
*   **Accuracy (%):** Used for multiple-choice questions in the AGIEval benchmark (2309.11235.pdf, p. 6).

## Evaluation Data
This section provides details about the datasets used to evaluate the model.

### Datasets:
The model was evaluated on four standard, publicly available benchmarks:
*   **AlpacaEval:** A benchmark for instruction-following that includes a test set of 805 user instructions from diverse sources (2309.11235.pdf, p. 15, Appendix C).
*   **MT-bench:** A multi-turn benchmark with 80 high-quality questions covering eight topics: writing, roleplay, extraction, reasoning, math, coding, and two knowledge areas (STEM and humanities) (2309.11235.pdf, p. 15, Appendix C).
*   **Vicuna-bench:** A benchmark that evaluates proficiency across eight question categories, including generic, knowledge, roleplay, commonsense, and coding (2309.11235.pdf, p. 15, Appendix C).
*   **AGIEval:** A collection of human-centric standardized tests, including English multiple-choice tasks from general college admission (SAT), law school admission (LSAT), and civil service exams (LogiQA) (2309.11235.pdf, p. 15, Appendix C).

### Motivation:
These datasets were chosen because they are widely recognized benchmarks for assessing two key capabilities of LLMs: instruction-following and generalization to complex, human-centric tasks (2309.11235.pdf, p. 5).

### Preprocessing:
For evaluation, the official zero-shot prompt and answer matching procedures for each benchmark were used. For conversational models, the benchmark questions were formatted as the user's turn within the model's specific conversation template (2309.11235.pdf, p. 16).

## Training Data
This section provides details about the datasets used to train the model.

### Datasets:
The model was fine-tuned on the **ShareGPT dataset**, a publicly available collection of user-shared conversations with ChatGPT (2309.11235.pdf, p. 5).
*   **Size:** Approximately 70,000 conversations (2309.11235.pdf, p. 5).
*   **Structure and Diversity:** The dataset is composed of mixed-quality data, which was partitioned into two classes based on the source model:
    *   **Expert Data (`D_exp`):** ~6,000 conversations generated by GPT-4.
    *   **Sub-optimal Data (`D_sub`):** The remaining ~64,000 conversations generated by GPT-3.5 (2309.11235.pdf, p. 5).

### Motivation:
The ShareGPT dataset was chosen because it exemplifies the common scenario of having a training dataset with mixed-quality data. The goal of the OpenChat project was to develop a method that could effectively learn from such data without requiring costly human preference labels, making this dataset highly suitable (2309.11235.pdf, pp. 1, 3).

### Preprocessing:
The training data was preprocessed to create a class-conditioned dataset.
1.  Conversations were separated into two groups based on their source (GPT-4 or GPT-3.5) (2309.11235.pdf, p. 4).
2.  Each group was formatted using a distinct conversation template to explicitly signal the data source to the model. For example:
    *   **GPT-4 Template:** `GPT4 User: {question}<|end_of_turn|>GPT4 Assistant:`
    *   **GPT-3.5 Template:** `GPT3 User: {question}<|end_of_turn|>GPT3 Assistant:`
3.  A special token, `<|end_of_turn|>`, was introduced at the end of each utterance to differentiate speakers (2309.11235.pdf, p. 4).

## Quantitative Analyses
This section presents disaggregated evaluation results.

### Unitary results:
The model's performance was benchmarked against other popular open-source and API-based models. On average, `openchat-13b` achieved the highest performance among all 13B open-source models (2309.11235.pdf, p. 1, Table 1 on p. 6).
*   **AlpacaEval:** 89.5% win rate.
*   **MT-bench:** 57.5 score.
*   **Vicuna-bench:** 85.0% win rate.
*   **Average Win Rate:** 77.3%.

Disaggregated results on AGIEval tasks are also provided, with `openchat-13b` achieving an average accuracy of 36.4%, outperforming the Llama-2-13B base model and other fine-tuned versions (2309.11235.pdf, p. 16, Table 5).

### Intersectional results:
Ablation studies were performed to analyze the impact of the core components of the C-RLFT method and the composition of the training data (2309.11235.pdf, p. 8, Table 2):
*   **Without Coarse-Grained Rewards:** Average performance drops from 77.3% to 74.0%.
*   **Without Class-Conditioning:** Average performance drops significantly from 77.3% to 60.5%.
*   **Training on GPT-4 data only (~6k samples):** Achieves a high average score of 64.5%, indicating the importance of data quality over quantity.
*   **Training on GPT-3.5 data only (~64k samples):** Achieves an average score of 42.8%.

The paper also analyzed the effect of varying the proportion of GPT-4 and GPT-3.5 data, demonstrating that expert data is extremely important even in limited quantities (2309.11235.pdf, p. 9, Figure 6).

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
The authors acknowledge several limitations and areas for future improvement (2309.11235.pdf, p. 10):
*   **Oversimplified Reward Scheme:** The assumption of data quality based solely on the source model (GPT-4 vs. GPT-3.5) may be "overly simplistic." The coarse-grained rewards could be more finely tuned to reflect the actual quality of each data point.
*   **Focus on Instruction-Following:** The model primarily focuses on enhancing instruction-following capabilities. Its reasoning abilities were not a primary target for improvement and represent an area for future work.
*   **Inference Prompt Sensitivity:** The model's performance is sensitive to the prompt template used during inference. Using a template associated with lower-quality data during training will result in a substantial performance decline (2309.11235.pdf, p. 8, Figure 5).

### Recommendations:
*   **Use the Correct Inference Prompt:** To achieve the highest quality responses, users should use the "GPT-4" class-conditioned prompt template during inference: `GPT4 User: {question}<|end_of_turn|>GPT4 Assistant:` (2309.11235.pdf, p. 8).
