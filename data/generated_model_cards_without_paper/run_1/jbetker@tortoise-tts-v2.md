### Model type:
Tortoise TTS is a multi-voice text-to-speech system composed of several models working in concert (api.py). The core components are:

1.  **Autoregressive Model (`UnifiedVoice`)**: A 30-layer GPT-2-like transformer model that takes text and conditioning latents and predicts discrete audio codes (autoregressive.py, api.py).
    *   **Architecture**: Transformer with 30 layers, 1024 model dimensions, and 16 heads (api.py).
    *   **Context Length**: Supports a maximum of 604 MEL tokens and 402 text tokens (api.py).
    *   **Tokens**: Uses 255 text tokens and a vocabulary of 8194 MEL codes (api.py, autoregressive.py).

2.  **CLVP Model (`CLVP`)**: A Contrastive Language-Voice Pre-training model used to filter the outputs of the autoregressive model. It scores how well the generated audio matches the input text (clvp.py, api.py).
    *   **Architecture**: Consists of two transformer encoders, one for text and one for speech (clvp.py).
    *   **Text Encoder**: 12 layers, 8 heads, 512 dimensions, for sequences up to 350 tokens (api.py).
    *   **Speech Encoder**: 12 layers, 8 heads, 512 dimensions, for sequences up to 430 tokens (api.py).

3.  **CVVP Model (`CVVP`)**: A Contrastive Voice-Voice Pre-training model, similar to CLVP, but it scores how well the generated audio matches the reference voice clips (cvvp.py, api.py).
    *   **Architecture**: Transformer-based model with a conditioning encoder (8 layers) and a speech encoder (8 layers) (api.py).
    *   **Parameters**: 512 model dimensions, 8 transformer heads (api.py).

4.  **Diffusion Decoder (`DiffusionTts`)**: A diffusion model that converts the discrete audio codes from the autoregressive model into a MEL spectrogram (diffusion_decoder.py, api.py).
    *   **Architecture**: A 10-layer U-Net-like model with attention blocks (diffusion_decoder.py, api.py).
    *   **Parameters**: 1024 model channels, 16 attention heads (api.py).

5.  **Vocoder (`UnivNetGenerator`)**: A UnivNet vocoder that converts the MEL spectrogram into a final audio waveform (vocoder.py, api.py).

The complete system also includes a `ConditioningEncoder` to produce latents from voice samples and a classifier (`AudioMiniEncoderWithClassifierHead`) to detect if an audio clip was generated by Tortoise (autoregressive.py, classifier.py, api.py).

### Training details:
The model uses a combination of supervised learning and contrastive learning. The overall process involves generating multiple audio candidates with the autoregressive model and then using the CLVP and CVVP models to select the best ones (api.py). The selected audio codes are then converted to a spectrogram by the diffusion decoder and finally to a waveform by the vocoder (api.py).

Key inference parameters that can be adjusted include:
*   **`num_autoregressive_samples`**: The number of candidate audio clips generated by the autoregressive model (api.py).
*   **`temperature`**: Softmax temperature for the autoregressive model (api.py).
*   **`top_p`**: Nucleus sampling p-value (api.py).
*   **`diffusion_iterations`**: Number of steps for the diffusion decoder to refine the output spectrogram (api.py).
*   **`cond_free`**: A boolean that enables or disables conditioning-free diffusion, which is reported to dramatically improve realism (api.py).
*   **`clvp_cvvp_slider`**: A value between 0 and 1 that balances the influence of the CLVP (text-matching) and CVVP (voice-matching) models in selecting the best candidate (api.py).

A hyperparameter sweep script in the repository explores variations of `top_p`, `temperature`, `diffusion_temperature`, and `cond_free_k` (sweep.py).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The primary use of Tortoise-TTS is to generate speech from text (Text-to-Speech). Its key capabilities include:

*   **Voice Cloning**: Generating speech that mimics the voice from a few reference audio clips (approximately 10 seconds each) (api.py). The input is text and a list of voice samples, and the output is a 24kHz audio waveform (api.py).
*   **Voice Combination**: Creating a new voice by combining traits from multiple reference voices (tortoise_tts.ipynb.txt).
*   **Random Voice Generation**: Generating speech with a random voice without needing any reference clips (tortoise_tts.ipynb.txt).
*   **Long-form Reading**: The model can be used to read long text files by splitting the text into smaller segments and generating audio for each part (read.py).

Example audio outputs are provided in the repository, demonstrating the model's ability to generate speech for various voices and texts (tortoise_v2_examples.html).

### Primary intended users:
The model is intended for developers and researchers interested in high-quality, multi-voice text-to-speech synthesis (setup.py, tortoise_tts.ipynb.txt).

(Source: tortoise_tts.ipynb.txt)

Here is a basic example of how to use the model to generate speech from text using a pre-packaged voice:
```python
# Imports used through the rest of the notebook.
import torch
import torchaudio
import torch.nn as nn
import torch.nn.functional as F

import IPython

from tortoise.api import TextToSpeech
from tortoise.utils.audio import load_audio, load_voice, load_voices

# This will download all the models used by Tortoise from the HF hub.
tts = TextToSpeech()

# This is the text that will be spoken.
text = "Joining two modalities results in a surprising increase in generalization! What would happen if we combined them all?"

# Pick a "preset mode" to determine quality. 
# Options: {"ultra_fast", "fast", "standard", "high_quality"}.
preset = "fast"

# Pick one of the available voices.
voice = 'tom'

# Load the voice samples and conditioning latents.
voice_samples, conditioning_latents = load_voice(voice)

# Generate speech.
gen = tts.tts_with_preset(text, voice_samples=voice_samples, conditioning_latents=conditioning_latents, 
                          preset=preset)
torchaudio.save('generated.wav', gen.squeeze(0).cpu(), 24000)
IPython.display.Audio('generated.wav')
```
(Source: tortoise_tts.ipynb.txt)

The model offers several presets that balance quality and inference speed:
*   **`ultra_fast`**: Fastest generation with lower quality.
*   **`fast`**: A good balance of quality and speed, suitable for mass inference.
*   **`standard`**: Very good quality.
*   **`high_quality`**: The best possible quality, but at a significant computational cost.
(Source: api.py)

The repository also includes scripts for command-line usage, such as `do_tts.py` for generating a single clip and `read.py` for reading an entire text file (do_tts.py, read.py).

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
*   **Reference Voice Quality**: The quality and characteristics of the input `voice_samples` directly influence the generated voice.
*   **Text vs. Voice Similarity**: The `clvp_cvvp_slider` parameter allows users to balance the trade-off between how closely the generated audio follows the text versus how similar it sounds to the reference voice (api.py). A value closer to 1 prioritizes text intelligibility, while a value closer to 0 prioritizes voice similarity (do_tts.py).

This section describes how the model's performance is evaluated.

### Model performance measures:
The repository does not specify the final quantitative metrics used to evaluate the model. However, the architecture relies on two models, CLVP and CVVP, to internally score and filter the generated audio candidates. These models compute similarity scores between the generated audio and the input text (CLVP) and between the generated audio and the reference voice clips (CVVP) (api.py).

### Decision thresholds:
During inference, the model generates `num_autoregressive_samples` candidates and then uses the CLVP and CVVP scores to select the top `k` best results. This `top_k` selection acts as a decision threshold (api.py).

## Ethical Considerations
This section discusses the ethical considerations in model development, including challenges, risks, and solutions.

The ability to clone voices with high fidelity presents ethical risks, such as the potential for creating convincing deepfakes for malicious purposes. The developers have included a tool to mitigate this risk.

*   **Misuse Detection**: The repository includes a script, `is_this_from_tortoise.py`, which provides a classifier model (`AudioMiniEncoderWithClassifierHead`) to determine the probability that a given audio clip was generated by Tortoise. This can be used to identify synthetic media created with this tool (is_this_from_tortoise.py, tortoise/api.py).
*   **Redaction**: The API includes an `enable_redaction` feature. When enabled, text enclosed in brackets `[]` is redacted from the final spoken output but is still used by the model for generation. This can be used for prompt engineering without leaking sensitive information in the audio (tortoise/api.py).
*   **Liability**: The Apache 2.0 license under which the model is released includes a "Limitation of Liability" clause, stating that contributors are not liable for damages arising from the use of the software (LICENSE.txt).

No information is provided regarding the use of sensitive data during training.

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
*   **Performance with Random Voices**: The developers note that "random voices can be prone to strange utterances" (tortoise_tts.ipynb.txt).
*   **Output Artifacts**: The `fix_autoregressive_output` function in `api.py` contains a comment explaining that failing to apply specific padding to the generated audio codes can result in speech with a "harsh end that sounds like 'BLAH' or similar" (api.py).
*   **Long Text Issues**: The model may fail to generate speech for very long text inputs. A warning is printed during inference if no stop tokens are found, which "typically means the spoken audio is too long" (api.py).

### Recommendations:
*   **Use a GPU**: For reasonable inference times, a GPU is strongly recommended (tortoise_tts.ipynb.txt).
*   **Handling Long Texts**: For long-form audio generation, it is recommended to split the text into smaller segments. The `read.py` script provides a `split_and_recombine_text` function for this purpose (read.py).
*   **Use Presets**: For general use, the provided presets (`ultra_fast`, `fast`, `standard`, `high_quality`) offer a good starting point for balancing generation speed and audio quality (api.py).
