## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by Sanchit Gandhi, Patrick von Platen, and Alexander M. Rush from Hugging Face (2311.00430.pdf, p. 1).

### Model date:
The academic paper describing the model was submitted on November 1, 2023 (2311.00430.pdf, p. 1).

### Model version:
This model is `distil-large-v2`, a distilled version of the Whisper `large-v2` model. It is 5.8 times faster and has 51% fewer parameters than the original `large-v2` model, while performing to within 1% Word Error Rate (WER) on out-of-distribution test data (2311.00430.pdf, p. 1). The model has 756 million parameters (2311.00430.pdf, p. 8, Table 5). Another version, `distil-medium.en`, was also developed, which is a distilled version of the `medium.en` model (2311.00430.pdf, p. 6).

### Model type:
The model is a sequence-to-sequence (Seq2Seq) Transformer-based model for Automatic Speech Recognition (ASR), specifically a distilled version of the Whisper model (2311.00430.pdf, p. 3). The architecture is named `WhisperForConditionalGeneration` (config.json).

**Architecture Details:**
*   **Model Type:** Whisper (config.json).
*   **Encoder-Decoder:** The model is an encoder-decoder architecture (config.json).
*   **Encoder:** It uses the full 32-layer encoder from the `large-v2` teacher model, which is frozen during training. The encoder has a model dimension (`d_model`) of 1280, 20 attention heads, and a feed-forward network dimension (`encoder_ffn_dim`) of 5120 (2311.00430.pdf, p. 6; config.json).
*   **Decoder:** The decoder consists of only 2 layers, initialized from the first and last layers of the 32-layer teacher decoder. It has a model dimension (`d_model`) of 1280, 20 attention heads, and a feed-forward network dimension (`decoder_ffn_dim`) of 5120 (2311.00430.pdf, p. 6, Figure 1; config.json).
*   **Parameters:** The model has 756 million parameters (2311.00430.pdf, p. 8, Table 5).
*   **Vocabulary Size:** 51,865 (config.json).
*   **Audio Features:** The model processes audio by converting it into 80-bin Mel spectrograms (config.json). The feature extractor uses a sampling rate of 16000 Hz and a hop length of 160 (preprocessor_config.json).

### Training details:
The model was trained using knowledge distillation (KD), a compression technique where a smaller "student" model (Distil-Whisper) is trained to reproduce the behavior of a larger "teacher" model (Whisper large-v2) (2311.00430.pdf, p. 3).

**Training Objective:**
The training objective is a weighted sum of two loss terms:
1.  **Pseudo-Labelling (PL) Loss:** A sequence-level KD loss where the student model is trained on pseudo-labels generated by the teacher model. This is a cross-entropy loss (2311.00430.pdf, p. 4).
2.  **Kullback-Leibler (KL) Divergence Loss:** A word-level KD loss where the student model's full probability distribution is trained to match the teacher's distribution by minimizing the KL divergence (2311.00430.pdf, p. 4).
The final objective is `L_KD = 0.8 * L_KL + 1.0 * L_PL` (2311.00430.pdf, p. 4).

**Key Training Parameters:**
*   **Frameworks:** JAX and Flax (2311.00430.pdf, p. 22).
*   **Optimizer:** AdamW with β1=0.9 and β2=0.999 (2311.00430.pdf, p. 23, Table 15).
*   **Learning Rate Schedule:** A slanted triangular learning rate (STLR) schedule was used, with a linear warmup for the first 500 steps to a maximum of 1e-4, followed by a linear decay to zero (2311.00430.pdf, p. 23).
*   **Batch Size:** 256 (2311.00430.pdf, p. 23, Table 15).
*   **Training Steps:** 80,000 optimization steps, equivalent to eight epochs of training (2311.00430.pdf, p. 6, 23).
*   **Precision:** bfloat16 (2311.00430.pdf, p. 23, Table 15).
*   **KL Softmax Temperature:** 2.0 (2311.00430.pdf, p. 23, Table 15).

**Methodology:**
*   The encoder from the teacher model (`large-v2`) was copied entirely and frozen during training (2311.00430.pdf, p. 6).
*   The 2-layer student decoder was initialized by copying the weights from the first and last layers of the teacher's 32-layer decoder (2311.00430.pdf, p. 6).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The model is intended for Automatic Speech Recognition (ASR) for the English language (2311.00430.pdf, p. 3; config.json). It is specifically designed for use in low-latency or resource-constrained environments where running larger models is challenging (2311.00430.pdf, p. 1).

**Capabilities:**
*   **Speech-to-Text Transcription:** The model transcribes spoken English audio into text (2311.00430.pdf, p. 1). It supports both short-form and long-form audio transcription (2311.00430.pdf, p. 5).
*   **Robustness:** It maintains robustness to difficult acoustic conditions and various audio domains (e.g., interviews, podcasts, telephone conversations) (2311.00430.pdf, p. 2).
*   **Speculative Decoding:** It is designed to be used as an "assistant model" with the larger Whisper model for speculative decoding, which can accelerate inference speed by up to 2 times while ensuring the output is identical to the original model (2311.00430.pdf, p. 1, 5).
*   **Language and Task:** The model is forced to predict English (`<|en|>`) transcription (`<|transcribe|>`) without timestamps (`<|notimestamps|>`) by default (config.json, `forced_decoder_ids`).

### Primary intended users:
The primary users are developers and researchers in the field of speech recognition who require a smaller, faster alternative to the large Whisper model without a significant loss in performance, especially for deployment in resource-constrained settings (2311.00430.pdf, p. 1).

### Out-of-scope uses:
*   **Non-English Languages:** This model is distilled from English-only checkpoints (`medium.en`, `large-v2`) and trained on English data. It is not intended for transcribing other languages (2311.00430.pdf, p. 6; config.json). The tokenizer includes tokens for other languages, but the model's training was English-specific (merges.txt, vocab.json).
*   **Translation:** The model is designed for transcription (`<|transcribe|>`) and not for speech translation (`<|translate|>`) (config.json, `forced_decoder_ids`).
*   **Timestamp Prediction:** The model is explicitly configured not to predict timestamps (`<|notimestamps|>`) (config.json, `forced_decoder_ids`).

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
*   **Acoustic Conditions:** The model's performance is influenced by the acoustic environment, including background noise. The developers note that performance degrades as noise becomes more intensive (2311.00430.pdf, p. 9).
*   **Audio Domain:** The model's robustness is evaluated across 10 distinct domains, including government interviews, podcasts, telephone conversations, and financial meetings, indicating that the domain of the audio is a relevant factor (2311.00430.pdf, p. 2, 21).
*   **Speaker Diversity:** The training data encompasses over 18,260 speakers, suggesting that speaker characteristics (e.g., accent, speaking style) are relevant to performance (2311.00430.pdf, p. 20, Table 13).
*   **Audio Length:** The model's behavior, particularly its propensity for hallucinations, is more prevalent in long-form audio transcription, especially with long silences (2311.00430.pdf, p. 8).

### Evaluation factors:
*   **In-Distribution (ID) vs. Out-of-Distribution (OOD):** The model is evaluated on both ID datasets (test splits of the training data) and OOD datasets (data not seen during training) to measure its generalization ability (2311.00430.pdf, p. 7).
*   **Noise Levels:** Performance is explicitly evaluated on the LibriSpeech test-clean dataset with varying levels of additive white noise and pub noise, measured by Signal-to-Noise Ratio (SNR) (2311.00430.pdf, p. 7, 9).
*   **Audio Form:** The model is evaluated separately on short-form (utterances up to 30 seconds) and long-form (minutes or hours) audio (2311.00430.pdf, p. 7, 8).

## Metrics
This section describes how the model's performance is evaluated.

### Model performance measures:
*   **Word Error Rate (WER):** This is the primary metric used to assess transcription accuracy. It is calculated after standardizing text with the Whisper English normaliser to focus on transcription errors rather than formatting differences (2311.00430.pdf, p. 23).
*   **Relative Error Rate (RER):** Used to measure "effective robustness" by comparing the performance on an OOD dataset to a reference ID dataset (2311.00430.pdf, p. 7, 9).
*   **Long-Form Metrics:** For long-form audio, the following metrics are used to quantify specific error types (2311.00430.pdf, p. 8):
    *   **Repeated 5-gram Word Duplicates (5-Dup.):** Measures the number of repetitive hallucinations.
    *   **Insertion Error Rate (IER):** Quantifies the frequency of hallucinations (predicting text not present in the audio).
    *   **Substitution Error Rate (SER):** Quantifies the frequency of substitution errors.
    *   **Deletion Error Rate (DER):** Quantifies the frequency of deletion errors.
*   **Relative Latency / Real-Time Factor (RTF):** Used to measure inference speed relative to the teacher model or the audio duration (2311.00430.pdf, p. 8, 29).

### Decision thresholds:
A Word Error Rate (WER) threshold (`λ`) was used during the data preparation phase to filter the pseudo-labelled training data. Only samples where the WER between the normalized ground truth and the normalized pseudo-label was below this threshold were used for training. A threshold of 10% was found to provide a good trade-off between data quantity and quality (2311.00430.pdf, p. 4, 11).

### Variation approaches:
Performance is reported as macro-averages across multiple datasets to provide a comprehensive measure of overall and effective robustness. For instance, average WER is calculated across 11 in-distribution datasets and 4 out-of-distribution datasets separately (2311.00430.pdf, p. 7, 24).

## Evaluation Data
This section provides details about the datasets used to evaluate the model.

### Datasets:
The model was evaluated on a total of 15 short-form and 4 long-form datasets.

**Short-Form Datasets (2311.00430.pdf, p. 24, Table 16):**
*   **In-Distribution (ID):** Test splits of the 11 training datasets: AMI IHM, AMI SDM, Call Home, Common Voice 13, GigaSpeech, LibriSpeech clean, LibriSpeech other, People's Speech, SwitchBoard, TED-LIUM, and Voxpopuli.
*   **Out-of-Distribution (OOD):**
    *   **CHIME-4:** 7 hours, 87 speakers, news broadcast domain (2311.00430.pdf, p. 7, Table 4).
    *   **Earnings-22:** 115 hours, unknown speakers, financial meetings domain (2311.00430.pdf, p. 7, Table 4).
    *   **FLEURS (en_us subset):** 2 hours, 3 speakers, narrated Wikipedia domain (2311.00430.pdf, p. 7, Table 4, 22).
    *   **SPGISpeech:** 100 hours, unknown speakers, financial meetings domain (2311.00430.pdf, p. 7, Table 4).

**Long-Form Datasets (OOD) (2311.00430.pdf, p. 7, Table 4):**
*   **Earnings-21:** 39 hours, financial meetings domain.
*   **Earnings-22:** 115 hours, financial meetings domain.
*   **Meanwhile:** 1 hour, 1 speaker, TV show domain.
*   **Rev 16:** 16 hours, podcast domain.

### Motivation:
The datasets were chosen to cover a broad range of acoustic conditions, speakers, speaking styles, and domains. The inclusion of both in-distribution (ID) and out-of-distribution (OOD) datasets was motivated by the need to measure the model's ability to generalize and maintain the robustness of the original Whisper model (2311.00430.pdf, p. 7). Long-form datasets were used to specifically evaluate performance on real-world applications like meeting transcription and to analyze the model's susceptibility to hallucinations (2311.00430.pdf, p. 5, 8).

### Preprocessing:
For evaluation, all transcriptions (both model predictions and ground truth) are standardized using the Whisper English normaliser. This process involves managing whitespace, converting numeric expressions, and removing or converting specific words and symbols to ensure that the WER metric penalizes actual transcription errors rather than formatting differences (2311.00430.pdf, p. 23). For long-form evaluation, a chunked transcription algorithm is used, which splits long audio files into smaller overlapping segments for parallel processing (2311.00430.pdf, p. 5).

## Training Data
This section provides details about the datasets used to train the model.

### Datasets:
The model was trained on a large-scale corpus assembled from nine publicly available English speech recognition datasets, totaling 21,170 hours of audio from over 18,260 speakers (2311.00430.pdf, p. 5, 20). The datasets cover 10 distinct domains (2311.00430.pdf, p. 2).

The combined training datasets include (2311.00430.pdf, p. 6, Table 2; p. 20, Table 13):
*   **People's Speech:** 12,000 hours (Government, interviews)
*   **GigaSpeech:** 2,500 hours (Audiobook, podcast, YouTube)
*   **Common Voice 13:** 2,400 hours (Narrated Wikipedia)
*   **Fisher:** 1,960 hours (Telephone conversations)
*   **LibriSpeech:** 960 hours (Audiobooks)
*   **VoxPopuli:** 540 hours (European Parliament)
*   **TED-LIUM:** 450 hours (TED talks)
*   **SwitchBoard:** 260 hours (Telephone conversations)
*   **AMI:** 100 hours (Meetings)

### Motivation:
The datasets were combined to create a large and diverse corpus for large-scale knowledge distillation, inspired by the SpeechStew methodology. The goal was to ensure the distilled model maintains robustness across different acoustic conditions and domains, similar to the original Whisper model which was trained on a massive, noisy web-scraped dataset (2311.00430.pdf, p. 3, 5).

### Preprocessing:
The training process used pseudo-labelling. The entire 21,170-hour training corpus was transcribed by the teacher model (Whisper large-v2) using greedy decoding to generate pseudo-labels (2311.00430.pdf, p. 5).

A filtering heuristic was then applied:
1.  Both the original ground-truth labels and the generated pseudo-labels were normalized using the Whisper English normaliser (2311.00430.pdf, p. 4).
2.  The Word Error Rate (WER) was computed between the normalized ground truth and the normalized pseudo-label for each sample (2311.00430.pdf, p. 4).
3.  Samples with a WER exceeding a threshold of 10% were discarded to ensure only high-quality pseudo-labels were used for training. This filtering removed 45.4% of the initial data (2311.00430.pdf, p. 4, 12).

## Quantitative Analyses
This section presents disaggregated evaluation results.

### Unitary results:
**Short-Form Evaluation (Average WER on 4 OOD datasets) (2311.00430.pdf, p. 8, Table 5):**
*   **distil-large-v2:** 10.1% WER
*   **large-v2 (teacher):** 9.1% WER
*   **distil-medium.en:** 11.1% WER
*   **medium.en:** 9.5% WER

**Long-Form Evaluation (Average WER on 4 OOD datasets) (2311.00430.pdf, p. 8, Table 5):**
*   **distil-large-v2:** 11.6% WER
*   **large-v2 (teacher):** 11.7% WER
*   **distil-medium.en:** 12.4% WER
*   **medium.en:** 12.3% WER

**Long-Form Error Analysis (distil-large-v2 vs. large-v2) (2311.00430.pdf, p. 11, Table 8):**
*   **Repetitions (5-Dup.):** 18503 (distil-large-v2) vs. 23792 (large-v2)
*   **Insertion Error Rate (IER):** 2.1% (distil-large-v2) vs. 3.3% (large-v2)
*   **Substitution Error Rate (SER):** 5.3% (distil-large-v2) vs. 3.9% (large-v2)
*   **Deletion Error Rate (DER):** 4.2% (distil-large-v2) vs. 4.5% (large-v2)

### Intersectional results:
**Performance under Additive Noise (on LibriSpeech test-clean) (2311.00430.pdf, p. 9, Figure 2):**
The WER performance of Distil-Whisper degrades less severely under intense noise compared to models trained only on LibriSpeech. Its noise robustness is similar to the teacher Whisper model, especially under naturalistic "pub noise" below 10 dB SNR, as they share the same frozen encoder.

**Effective Robustness (Relative Error Rate vs. GigaSpeech) (2311.00430.pdf, p. 9, Table 6):**
*   **CHIME-4:** 18.4% RER (performs worse than baseline)
*   **Earnings-22:** 1.6% RER (performs worse than baseline)
*   **FLEURS:** 48.2% RER (performs worse than baseline)
*   **SPGISpeech:** -12.8% RER (performs better than baseline)
*   **Average RER:** 10.7% (overall, a 0.8% absolute WER increase compared to the teacher model on OOD data).

## Memory or Hardware Requirements
This section outlines the memory or hardware requirements for loading, deploying, and training the model.

### Loading Requirements:
The model has 756 million parameters. The encoder can be shared with the main Whisper model when used for speculative decoding, requiring only the distilled decoder layers to be loaded in addition (2311.00430.pdf, p. 10).

### Deploying Requirements:
Inference latency was benchmarked on single GPU setups.
*   **A100 40GB GPU (with PyTorch 2.0 and Flash Attention 2):**
    *   Batch size 1 (short-form): Real-time factor (RTF) is 24.2 x 10⁻³ (2311.00430.pdf, p. 30, Table 24). This is 5.8 times faster than the `large-v2` model (2311.00430.pdf, p. 8, Table 5).
    *   Batch size 16 (long-form): RTF is 4.4 x 10⁻³ (2311.00430.pdf, p. 30, Table 24).
*   **T4 16GB GPU (with PyTorch 2.0 and Flash Attention 1):**
    *   Batch size 1: RTF is 27.4 x 10⁻³ (2311.00430.pdf, p. 30, Table 25).
    *   Batch size 16: RTF is 20.1 x 10⁻³ (2311.00430.pdf, p. 30, Table 25).

### Training or Fine-tuning Requirements:
The model was trained using data parallelism across TPU v4-8 accelerators with bfloat16 precision (2311.00430.pdf, p. 22).

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
*   **Substitution Errors:** While the model shows improvement in insertion errors (hallucinations) on long-form audio, it is subject to more substitution errors (SER) compared to the teacher `large-v2` model (1.4% higher SER) (2311.00430.pdf, p. 10).
*   **Performance Trade-off:** There is a clear trade-off between latency and performance. While this model is significantly faster, it performs approximately 1% worse on average on OOD short-form datasets compared to the `large-v2` model (2311.00430.pdf, p. 8).
*   **Encoder Distillation:** Jointly distilling the encoder and decoder leads to significant performance degradation. Reducing the encoder to 16 layers (from 32) resulted in a 3.1% absolute increase in OOD WER, suggesting a deep encoder is crucial for maintaining performance (2311.00430.pdf, p. 12).
*   **High-Frequency Uncommon Words:** The model's performance can be lower on datasets containing a single speaker with a high frequency of uncommon words, as seen in the "Meanwhile" dataset evaluation (2311.00430.pdf, p. 9).

### Recommendations:
*   **Speculative Decoding:** The model is recommended for use as an assistant model for speculative decoding with the `large-v2` model. This provides a 2x speed-up in inference while mathematically guaranteeing the same outputs as the original model (2311.00430.pdf, p. 11).
*   **Long-Form Transcription:** For long-form transcription using the chunked algorithm, a chunk length of 15 seconds is recommended for this model, as it yielded the best performance on the TED-LIUM validation set (4.1% WER) (2311.00430.pdf, p. 29, Table 23).
*   **Hardware:** For batch sizes greater than 1, higher memory GPUs like the A100 are preferential to avoid performance stagnation observed on lower-memory GPUs like the T4 (2311.00430.pdf, p. 30).
*   **Flash Attention:** Using Flash Attention 2 is recommended for higher batch sizes, as it improves inference time significantly more for Distil-Whisper than for the original Whisper model due to the architecture's deep encoder (2311.00430.pdf, p. 29).
