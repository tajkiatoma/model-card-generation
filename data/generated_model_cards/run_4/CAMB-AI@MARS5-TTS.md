### Model type:
MARS5 is a Text-to-Speech (TTS) model designed for voice cloning (inference.py; technical_details.md). It follows a two-stage Autoregressive (AR) and Non-Autoregressive (NAR) architecture (technical_details.md).

**Architecture Details:**
*   **Codec:** The model uses the Encodec model at a 24kHz sampling rate and a target bandwidth of 6.0 kbps to convert audio into discrete codes (inference.py).
*   **Tokenizers:** It employs two types of tokenizers:
    *   A `RegexTokenizer` with a GPT-4 style split pattern for processing input text (inference.py; mars5/minbpe/regex.py).
    *   A `CodebookTokenizer` for processing the discrete speech codes from Encodec (inference.py; mars5/minbpe/codebook.py).
*   **Autoregressive (AR) Component:** The AR model is a Mistral-style decoder-only transformer (`CodecLM`) with a dimension of 1536. It autoregressively predicts the first level (L0) of the 8 Encodec quantization codebooks from the input text and a speaker embedding (technical_details.md; mars5/model.py). The speaker embedding is derived from the reference audio using a small encoder-only transformer (technical_details.md).
*   **Non-Autoregressive (NAR) Component:** The NAR model is an encoder-decoder transformer (`ResidualTransformer`) with a dimension of 1024. It uses a multinomial diffusion framework (DDPM) to predict the remaining 7 Encodec codebooks based on the text, the speaker embedding, and the L0 codes generated by the AR model (technical_details.md; mars5/model.py; mars5/diffuser.py).
*   **Vocoder:** The final discrete codes are converted back into a waveform using a `Vocos` vocoder, specifically the `charactr/vocos-encodec-24khz` pretrained model (inference.py).

The model supports two cloning modes (technical_details.md; inference.py):
*   **Shallow Clone:** Uses the reference audio only to generate a speaker embedding. This mode is faster and does not require a transcript for the reference audio.
*   **Deep Clone:** Uses both the reference audio and its transcript as a prompt for the AR and NAR models. This improves the accuracy of speaker identity and prosody cloning but is slower and requires an accurate transcript.

### Training details:
*   **AR Model Training:** The autoregressive component was trained using a standard next-token prediction task with a cross-entropy loss. The loss on text tokens was given a smaller weight compared to speech tokens (technical_details.md).
*   **NAR Model Training:** The non-autoregressive component was trained using a multinomial diffusion framework. The diffusion process follows a cosine schedule. During training, the ground truth L0 codes are used to condition the prediction of the remaining 7 codebook levels (technical_details.md; mars5/diffuser.py).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The primary use of MARS5 is to perform Text-to-Speech (TTS) synthesis that clones the voice from a given reference audio clip (inference.py; technical_details.md).

**Capabilities:**
*   **Voice Cloning:** The model can generate speech in a voice similar to that in a reference audio sample (technical_details.md).
*   **Deep vs. Shallow Cloning:** It offers two modes for voice cloning:
    *   `deep_clone=True`: Provides higher-quality cloning of speaker identity and prosody but requires an accurate transcript of the reference audio and has higher latency (technical_details.md; inference.py).
    *   `deep_clone=False` (Shallow Clone): A faster method that does not require a reference transcript but may be less accurate in cloning (technical_details.md; inference.py).

**Input-Output Structure:**
*   **Input:** The `tts` function takes the following inputs (inference.py; handler.py):
    *   `text` (str): The text to be synthesized.
    *   `ref_audio` (torch.Tensor): A 1D tensor representing the reference audio waveform, sampled at 24kHz.
    *   `ref_transcript` (Optional[str]): The transcript of the reference audio, required for deep cloning.
    *   `cfg` (Optional[InferenceConfig]): A configuration object to control synthesis parameters like temperature, top_k, etc.
*   **Output:** The `tts` function returns a tuple containing (inference.py):
    *   `ar_codes` (torch.Tensor): A tensor of the discrete coarse codes generated by the AR model.
    *   `out_wav` (torch.Tensor): A 1D tensor of the final synthesized audio waveform, sampled at 24kHz.

The model is intended for developers and researchers working with speech synthesis and PyTorch (MARS5_TTS_Usage.ipynb).

## How to Use
This section outlines how to use the model.

The following code demonstrates how to load the MARS5 model and use it for text-to-speech synthesis with voice cloning (MARS5_TTS_Usage.ipynb).

**1. Installation:**
First, install the required dependencies.
```bash
pip install --upgrade vocos encodec librosa torch torchaudio numpy
```
(MARS5_TTS_Usage.ipynb; requirements.txt)

**2. Load the Model:**
Load the MARS5 model and its inference configuration class from Torch Hub.
```python
import torch
import librosa
import IPython.display as ipd

# Load the model and config class
mars5, config_class = torch.hub.load('Camb-ai/mars5-tts', 'mars5_english', trust_repo=True)
```
(MARS5_TTS_Usage.ipynb)

**3. Prepare Inputs:**
Load a reference audio file and provide its transcript if using deep clone. The audio should be a mono waveform sampled at the model's sampling rate (24kHz) (inference.py; MARS5_TTS_Usage.ipynb).

```python
# Load reference audio file
wav, sr = librosa.load('./example.wav', sr=mars5.sr, mono=True)
wav = torch.from_numpy(wav)

# Provide the transcript for the reference audio
ref_transcript = "We actually haven't managed to meet demand."

print("Reference audio:")
ipd.display(ipd.Audio(wav.numpy(), rate=mars5.sr))
print(f"Reference transcript: {ref_transcript}")
```
(MARS5_TTS_Usage.ipynb)

**4. Configure and Run Inference:**
Create an `InferenceConfig` object to customize the synthesis process. Set `deep_clone=True` for higher-quality cloning. Then, call the `tts` method.

```python
# Set deep_clone to True for better quality, or False for faster inference without a transcript.
deep_clone = True 

# Configure inference settings
cfg = config_class(deep_clone=deep_clone, rep_penalty_window=100,
                   top_k=100, temperature=0.7, freq_penalty=3)

# Text to synthesize
target_text = "The quick brown rat."

# Perform TTS
ar_codes, wav_out = mars5.tts(target_text, wav, 
                              ref_transcript,
                              cfg=cfg)

print('Synthesized output audio:')
ipd.display(ipd.Audio(wav_out.numpy(), rate=mars5.sr))
```
(MARS5_TTS_Usage.ipynb)

**Available Inference Settings:**
The `InferenceConfig` class provides numerous parameters to tune the output, including (inference.py):
*   `temperature`: Controls randomness.
*   `top_k`, `top_p`, `typical_p`: Nucleus and typical sampling parameters.
*   `freq_penalty`, `presence_penalty`: Penalties to control repetition.
*   `timesteps`: Number of diffusion steps for the NAR model.
*   `nar_guidance_w`: Classifier-free guidance weight for the NAR model.

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
*   **Reference Audio Quality:** The quality of the input reference audio directly impacts the quality of the cloned voice (technical_details.md).
*   **Reference Transcript Accuracy:** For deep cloning, an accurate transcript of the reference audio is required for best performance (inference.py; technical_details.md).
*   **Reference Audio Duration:** The model issues a warning if the reference audio is longer than the recommended maximum duration (`max_prompt_dur`, default 12 seconds), as this may cause quality degradation (inference.py).
*   **Inference Parameters:** The output is highly dependent on the settings in the `InferenceConfig` object, such as `temperature`, `top_k`, `top_p`, and various penalties (inference.py).
*   **Cloning Mode:** The choice between `deep_clone` and shallow clone affects the trade-off between cloning accuracy and inference speed (technical_details.md).

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
*   **Reference Audio Duration:** Using a reference audio clip longer than the recommended maximum (`max_prompt_dur`, default 12 seconds) may lead to degradations in output quality (inference.py).
*   **Generation Length:** If the model generates an output that reaches the maximum allowed length (`max_len`), it is a sign that the inference process likely failed or the input text was too long (ar_generate.py).
*   **Beam Search:** Only a `beam_width` of 1 is currently supported for autoregressive generation (inference.py; ar_generate.py).
*   **Deep Clone Requirement:** The `deep_clone` feature, which provides better quality, requires an accurate transcript of the reference audio. If a transcript is not provided while `deep_clone` is enabled, an error will occur (inference.py).

### Recommendations:
*   **Cloning Mode Selection:** For the most accurate cloning of a speaker's voice and prosody, use the `deep_clone=True` setting and provide an accurate transcript of the reference audio. For faster inference where a transcript is not available, use the shallow clone mode (`deep_clone=False`) (technical_details.md).
*   **Trimming Reference Audio:** It is recommended to trim the reference audio prompt to be shorter than the maximum prompt length (`max_prompt_dur`) to avoid quality issues (inference.py).
*   **Improving NAR Performance:** The quality of the non-autoregressive stage can be improved by using RePaint inpainting techniques. This can be enabled by increasing the `jump_len` and `jump_n_sample` parameters in the diffusion configuration, though this will increase inference time (technical_details.md).
*   **Silence Trimming:** The final output audio is automatically trimmed to remove leading and trailing silences using a threshold of `trim_db=27` (inference.py; mars5/trim.py). This can be adjusted in the `InferenceConfig`.
