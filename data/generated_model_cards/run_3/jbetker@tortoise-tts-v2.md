## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by James Betker (setup.py, CITATION.cff). The associated research paper describes the author as an independent researcher (paper.pdf, page 12).

### Model date:
The model was released on April 28, 2022 (CITATION.cff). The training process took place over a period of one year (paper.pdf, page 4).

### Model version:
This model card describes version 2.1.3 (setup.py). The accompanying paper refers to the model as "TorToise-v2" (paper.pdf, page 3).

### Model type:
TorToise is a multi-voice text-to-speech (TTS) system that leverages a combination of a transformer-based autoregressive model and a diffusion decoder to generate high-quality speech (paper.pdf, page 1). The overall architecture consists of several distinct neural networks that work in sequence (paper.pdf, page 3):

1.  **Autoregressive (AR) Model (`UnifiedVoice`):** This is a GPT-2 style transformer model that takes text and conditioning voice inputs and produces a sequence of discrete speech tokens and a latent vector.
    *   **Architecture:** Transformer stack with causal masking (paper.pdf, page 9).
    *   **Layers:** 30 (api.py, paper.pdf, page 9).
    *   **Model Dimension:** 1024 (api.py, paper.pdf, page 9).
    *   **Attention Heads:** 16 (api.py, paper.pdf, page 9).
    *   **Max Tokens:** 402 text tokens, 604 MEL tokens (api.py).

2.  **Contrastive Language-Voice Pretrained Transformer (CLVP) & Contrastive Voice-Voice Pretrained (CVVP):** These models are similar to CLIP and are used to rank the outputs of the autoregressive model. They calculate the correlation between the input text and the generated speech tokens, allowing the system to select the best candidates for the next stage (paper.pdf, page 4; api.py).
    *   **CLVP Architecture:** Dual transformer stacks (one for text, one for speech) (paper.pdf, page 10).
    *   **Depth:** 12 layers for text encoder, 12 for speech encoder (api.py). The paper mentions a depth of 20 for a similar model (paper.pdf, page 10).
    *   **Model Dimension:** 512 (api.py). The paper mentions a dimension of 768 (paper.pdf, page 10).
    *   **Attention Heads:** 8 (api.py). The paper mentions 12 heads (paper.pdf, page 10).

3.  **Diffusion Decoder (`DiffusionTts`):** A Denoising Diffusion Probabilistic Model (DDPM) that converts the speech tokens and latent vector from the AR model into a MEL spectrogram. It uses a bespoke architecture combining residual convolutions and dense self-attention (paper.pdf, pages 3, 11).
    *   **Architecture:** Alternating full attention and convolutional residual blocks (paper.pdf, page 12).
    *   **Depth:** 10 layers (api.py, paper.pdf, page 12).
    *   **Model Dimension:** 1024 (api.py, paper.pdf, page 12).
    *   **Attention Heads:** 16 (api.py, paper.pdf, page 12).

4.  **Vocoder (`UnivNetGenerator`):** A pre-trained UnivNet model is used as the final stage to convert the MEL spectrogram into a raw audio waveform at a 24kHz sample rate (paper.pdf, page 1; api.py).

5.  **Classifier (`AudioMiniEncoderWithClassifierHead`):** An additional model is provided to classify whether an audio clip was generated by Tortoise or is a real human recording (api.py, classify.py).

The pre-trained weights for these component models are downloaded from Hugging Face (api.py).

### Training details:
The model's training process involves several stages and techniques:

*   **Autoregressive Training:** The AR model is trained to predict speech tokens based on text and voice conditioning inputs. It uses a custom Byte Pair Encoding (BPE) tokenizer with 256 tokens (paper.pdf, page 9).
*   **Diffusion Training:** The diffusion model is trained to convert discrete speech codes into MEL spectrograms. A key technique, termed the "Tortoise Trick," involves fine-tuning the diffusion model on the latent space from the AR model's outputs, which is more semantically rich than discrete tokens and significantly improves output quality (paper.pdf, page 4).
*   **CLVP/CVVP Training:** The CLVP model is trained on pairs of audio clips and text in a contrastive setting to learn to discriminate between good and bad text-speech pairings (paper.pdf, page 4).
*   **Re-ranking:** During inference, the AR model generates a large number of candidate speech outputs. The CLVP model is then used to score these candidates, and only the top-k are passed to the more computationally expensive diffusion decoder. This re-ranking strategy is crucial for generating expressive outputs (paper.pdf, page 4).
*   **Inference Sampling:** For the autoregressive model, nucleus sampling is used with P=0.8, a repetition penalty of 2.0, and a softmax temperature of 0.8 (paper.pdf, page 4; api.py). For the diffusion model, the DDIM sampling algorithm is used with a linear schedule, 64 sampling steps, and a conditioning-free guidance constant of 2 (paper.pdf, page 5).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The primary use of Tortoise is for high-quality, multi-voice text-to-speech synthesis (setup.py). Its key capabilities include:

*   **Voice Cloning:** The model can generate speech in a target voice using just a few (~10 second) reference audio clips of that voice (api.py).
*   **Text-to-Speech:** It converts input text into spoken audio in the desired voice (tortoise_tts.py).
*   **Voice Combination:** The model can combine conditioning samples from multiple voices to create a new voice that shares traits from the parent voices (tortoise-tts.ipynb).

The model takes text and one or more 22.05kHz audio clips as input and produces a 24kHz audio waveform as output (api.py, tortoise_tts.py).

### Primary intended users:
The model is intended for researchers, developers, and hobbyists interested in speech synthesis. The availability of a Colab notebook makes it accessible to users without specialized hardware or deep technical expertise (tortoise-tts.ipynb).

### Out-of-scope uses:
The repository does not explicitly list out-of-scope uses. However, due to the model's high-fidelity voice cloning capabilities, any use for malicious purposes such as impersonation, creating deepfake audio for misinformation, or fraud is a potential misuse and should be considered out-of-scope. The model includes a classifier to detect generated audio, which can serve as a tool to mitigate such misuse (api.py, classify.py).

## How to Use
This section outlines how to use the model.

The model can be used via the `TextToSpeech` class in the `api.py` script. The repository provides several scripts and a Jupyter notebook demonstrating its usage.

**1. Installation:**
First, clone the repository and install the required dependencies.
```bash
!git clone https://github.com/neonbjb/tortoise-tts.git
%cd tortoise-tts
!pip3 install -r requirements.txt
!python3 setup.py install
```
(tortoise-tts.ipynb)

**2. Basic Usage with a Preset Voice:**
The model comes with pre-packaged voices. You can generate speech by providing text and the name of a voice.
```python
import torchaudio
import IPython
from tortoise.api import TextToSpeech
from tortoise.utils.audio import load_voice

# Initialize the TTS engine
tts = TextToSpeech()

# Define the text to be spoken
text = "Joining two modalities results in a surprising increase in generalization!"

# Select a voice
voice = 'tom'

# Load the voice samples and conditioning latents
voice_samples, conditioning_latents = load_voice(voice)

# Generate speech with a preset
preset = "fast"
gen = tts.tts_with_preset(text, voice_samples=voice_samples, conditioning_latents=conditioning_latents, preset=preset)

# Save and play the generated audio
torchaudio.save('generated.wav', gen.squeeze(0).cpu(), 24000)
IPython.display.Audio('generated.wav')
```
(tortoise-tts.ipynb)

**3. Voice Cloning with Custom Audio:**
You can provide your own audio clips to clone a voice. The clips should be 22.05kHz waveform data.
```python
from tortoise.api import TextToSpeech
from utils.audio import load_audio

tts = TextToSpeech()

# List of paths to your custom voice clips
cond_paths = ["path/to/your/clip1.wav", "path/to/your/clip2.wav"]
conds = []
for cond_path in cond_paths:
    c = load_audio(cond_path, 22050)
    conds.append(c)

# Generate speech
gen = tts.tts_with_preset("This is a test of my voice.", conds, preset='standard')
torchaudio.save('cloned_voice.wav', gen.squeeze(0).cpu(), 24000)
```
(tortoise_tts.py)

**4. Generation Presets:**
The model offers several presets that balance quality and inference speed:
*   `'ultra_fast'`: Fastest generation speed.
*   `'fast'`: Good quality at a decent speed.
*   `'standard'`: Very good quality.
*   `'high_quality'`: The best possible quality, at the cost of computation.
(api.py)

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
*   **Quality of Conditioning Audio:** The performance of voice cloning is highly dependent on the quality of the provided reference audio clips. The paper notes that the conditioning input allows the model to infer "vocal characteristics like tone and prosody" (paper.pdf, page 4). The training data pipeline included classifiers to remove clips with background noise, music, poor quality, multiple speakers, or reverb, indicating these factors are critical (paper.pdf, page 6).
*   **Generation Preset:** The choice of preset (`ultra_fast`, `fast`, `standard`, `high_quality`) directly impacts the trade-off between inference speed and output audio quality by changing the number of autoregressive samples and diffusion iterations (api.py).
*   **Generation Parameters:** Several parameters can be tuned to influence the output, such as `temperature`, `repetition_penalty`, and the `clvp_cvvp_slider`, which balances adherence to the text versus similarity to the reference voice (api.py).

### Evaluation factors:
The paper describes a custom evaluation suite that reports on the following factors:
*   **Sample Quality/Realism:** A CLVP-based distance metric, similar to the FID score in image generation, is used to compare the distance between real and generated samples (paper.pdf, page 5).
*   **Intelligibility:** An open-source wav2vec model is used to characterize the intelligibility of the generated speech segment (paper.pdf, page 5).

## Metrics
This section describes how the model's performance is evaluated.

### Model performance measures:
The performance of the model during training was measured using standard loss functions for each component:
*   **VQVAE:** Mean Squared Error (MSE) reconstruction loss and commitment loss (paper.pdf, page 7).
*   **Autoregressive Model:** Cross-entropy loss for predicting the next MEL token (paper.pdf, page 9).
*   **CLVP Model:** Contrastive loss (paper.pdf, page 10).
*   **Diffusion Model:** MSE and Variational Lower Bound (VLB) loss (paper.pdf, page 12).

For final evaluation, the author developed a custom suite that uses a CLVP-based distance metric for realism and a wav2vec model for intelligibility (paper.pdf, page 5).

### Decision thresholds:
During inference, the following thresholds and parameters are used:
*   **Nucleus Sampling:** The autoregressive model uses nucleus sampling with a probability threshold `top_p` of 0.8 (paper.pdf, page 4; api.py).
*   **CLVP/CVVP Slider:** The `clvp_cvvp_slider` parameter (ranging from 0 to 1) is used to balance the influence of the CLVP (text-speech similarity) and CVVP (voice similarity) models when selecting the best candidates. A value of 0.5 is used as a default in some scripts (api.py, tortoise_tts.py).

### Variation approaches:
The model uses a re-ranking approach to improve output quality. The autoregressive model generates a large number of candidate outputs (`num_autoregressive_samples`). These are then scored by the CLVP and CVVP models, and only the top `k` candidates are selected for the diffusion and vocoding steps. This allows the model to explore a wider output space and select the most plausible results (paper.pdf, page 4; api.py).

## Training Data
This section provides details about the datasets used to train the model.

### Datasets:
The model was trained on a large and diverse collection of speech data totaling approximately 50,000 hours. The datasets include:
1.  **LibriTTS:** A corpus of English speech derived from the LibriVox project, containing ~585 hours (paper.pdf, page 5).
2.  **HiFiTTS:** A multi-speaker English TTS dataset containing ~311 hours (paper.pdf, page 5).
3.  **Extended Dataset:** A custom-built dataset of 49,000 hours of audio scraped from audiobooks and podcasts on the internet (paper.pdf, pages 5-6).

### Motivation:
The developer's goal was to train a large-scale generative model, analogous to large language models in NLP, which necessitated a very large dataset to learn the complexities of human speech (paper.pdf, page 5).

### Preprocessing:
The extended dataset underwent a significant preprocessing pipeline:
*   Audio was split on 500ms silences, and only clips between 5 and 20 seconds were retained.
*   A series of custom-trained classifiers were used to filter the data, removing any clips containing background noise, music, poor quality (e.g., phone calls), multiple speakers, or reverb.
*   The cleaned audio clips were transcribed using a wav2vec2-large model that was fine-tuned to predict punctuation (commas, quotation marks, etc.), which is important for generating natural-sounding speech.
(paper.pdf, Appendix A, page 6)

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
The author identifies several limitations and areas for future improvement in the research paper:
*   **Limited Sequence Length:** The AR model uses fixed positional encodings, which limits the maximum length of speech it can generate. Longer sequences would require relative positional encodings (paper.pdf, page 12).
*   **CLVP Training Limitations:** The CLVP model was trained on audio clips of up to 13 seconds, which may be why its re-ranking performance suffers on longer samples (paper.pdf, page 12).
*   **Suboptimal Architecture:** The diffusion decoder's architecture omits feed-forward blocks, which the author retrospectively identifies as a "poor design decision" that should be corrected in future work (paper.pdf, page 12).
*   **Inconsistent Sample Rates:** The main model stack operates at 22kHz, but the diffusion decoder outputs MEL spectrograms computed from 24kHz audio to ensure compatibility with the pre-trained vocoder. This discrepancy was not for performance reasons and could be resolved by retraining the components at a consistent sample rate (paper.pdf, page 11).
*   **Under-trained Model:** The training curves suggest that the model was "far from overfitting," indicating that performance could likely be improved by simply training for longer on more data (paper.pdf, page 12).
*   **Inference Speed:** The model is computationally intensive and slow, which is why it is named "Tortoise" (tortoise-tts.ipynb).

### Recommendations:
*   **Use a GPU:** For reasonable inference times, a GPU is strongly recommended (tortoise-tts.ipynb).
*   **Use Presets:** Users should leverage the provided presets (`ultra_fast`, `fast`, `standard`, `high_quality`) to balance their needs for speed versus audio quality (api.py).
*   **Parameter Tuning:** For advanced users, experimenting with generation parameters like `temperature`, `repetition_penalty`, and `clvp_cvvp_slider` can help fine-tune the output for specific use cases (api.py).
*   **Ethical Use:** Users should be mindful of the ethical implications of voice cloning and refrain from using the model for malicious purposes such as creating non-consensual or deceptive content.
