## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by Sanchit Gandhi, Patrick von Platen, and Alexander M. Rush from Hugging Face (paper.pdf, Page 1).

### Model date:
The paper describing the model was submitted to arXiv on November 1, 2023 (paper.pdf, Page 1).

### Model version:
This model is named Distil-Whisper. It is a distilled version of OpenAI's Whisper model. The repository corresponds to the `distil-large-v2` checkpoint, which is a 2-layer decoder model distilled from the Whisper `large-v2` teacher model. The encoder is copied entirely from the `large-v2` teacher model and frozen during training, while the 2-layer decoder is initialized from the first and last layers of the teacher's decoder (paper.pdf, Page 6).

This distilled model is 5.8 times faster and has 51% fewer parameters than the original `large-v2` model, while performing to within 1% Word Error Rate (WER) on out-of-distribution test data (paper.pdf, Page 1). It is also less prone to hallucination errors on long-form audio compared to the original Whisper model (paper.pdf, Page 1).

### Model type:
Distil-Whisper is a sequence-to-sequence (Seq2Seq) Transformer model for Automatic Speech Recognition (ASR) (paper.pdf, Page 1, 3). It is an encoder-decoder model specifically of the "whisper" type (config.json).

**Architecture Details:**
*   **Overall Architecture:** `WhisperForConditionalGeneration` (config.json).
*   **Encoder:** The encoder has 32 layers (`encoder_layers`), a model dimension (`d_model`) of 1280, 20 attention heads (`encoder_attention_heads`), and a feed-forward network dimension (`encoder_ffn_dim`) of 5120. The entire encoder is copied from the `large-v2` teacher model and frozen during training (config.json; paper.pdf, Page 6).
*   **Decoder:** The decoder has 2 layers (`decoder_layers`), a model dimension (`d_model`) of 1280, 20 attention heads (`decoder_attention_heads`), and a feed-forward network dimension (`decoder_ffn_dim`) of 5120. The decoder layers are initialized from the first and last layers of the `large-v2` teacher's decoder (config.json; paper.pdf, Page 6).
*   **Activation Function:** GELU (`activation_function`) (config.json).
*   **Model Size:** The `distil-large-v2` model has 756 million parameters (paper.pdf, Page 7, Table 3).
*   **Vocabulary Size:** 51,865 (config.json).
*   **Supported Positions:** The model supports a maximum of 1500 source positions (30 seconds of audio) and 448 target positions (config.json).

### Training details:
The model was trained using knowledge distillation (KD), a compression technique where a smaller "student" model (Distil-Whisper) is trained to reproduce the behavior of a larger "teacher" model (Whisper large-v2) (paper.pdf, Page 3).

*   **Training Objective:** The training objective is a weighted sum of two loss terms:
    1.  **Pseudo-Labelling (PL) Loss:** A sequence-level cross-entropy loss where the student model is trained on pseudo-labels generated by the teacher model (paper.pdf, Page 4).
    2.  **Kullback-Leibler (KL) Divergence Loss:** A word-level loss where the student model's full probability distribution is trained to match the teacher's distribution over the vocabulary for each token (paper.pdf, Page 4).
    The weights for the KL and PL losses were set to 0.8 and 1.0, respectively (paper.pdf, Page 4).

*   **Training Procedure:**
    *   The model was trained for a total of 80,000 optimization steps, equivalent to eight epochs (paper.pdf, Page 6).
    *   **Optimizer:** AdamW with β1=0.9, β2=0.999, and a weight decay of 10e-8 (paper.pdf, Page 23, Table 15).
    *   **Learning Rate:** A slanted triangular learning rate (STLR) schedule was used, with a linear warmup from zero to a maximum of 1e-4 over the first 500 steps, followed by a linear decay to zero (paper.pdf, Page 23).
    *   **Batch Size:** A batch size of 256 was used (paper.pdf, Page 6).
    *   **Precision:** Training was performed with bfloat16 precision (paper.pdf, Page 22).
    *   **Regularization:** No data augmentation or standard regularization techniques were used. Instead, the model relies on the diversity of the large-scale training dataset for generalization (paper.pdf, Page 6). Gradient norm clipping was set to a max of 1.0 (paper.pdf, Page 23, Table 15).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
Distil-Whisper is intended for Automatic Speech Recognition (ASR) for the English language. Its primary advantage is providing robust transcription performance in low-latency or resource-constrained environments where larger models like Whisper `large-v2` are challenging to deploy (paper.pdf, Page 1).

Key capabilities include:
*   **Fast Transcription:** The model is 5.8 times faster than Whisper `large-v2` (paper.pdf, Page 1).
*   **Robustness:** It maintains robustness to different audio domains and noisy acoustic conditions, performing within 1% WER of the original Whisper model on out-of-distribution data (paper.pdf, Page 1, 2).
*   **Long-Form Transcription:** The model can be used with a chunked transcription algorithm for long audio files and is less prone to "hallucination" errors than the original Whisper model (paper.pdf, Page 1, 5).
*   **Speculative Decoding:** It is designed to be used as a faster "assistant" model with the original Whisper model for speculative decoding, which speeds up inference by a factor of 2 while guaranteeing the same output as the larger model (paper.pdf, Page 1, 5).

The model takes a 30-second audio input sampled at 16kHz and outputs the transcribed text (preprocessor_config.json; paper.pdf, Page 5).

### Primary intended users:
The primary users are developers and researchers in the field of speech recognition who require a smaller, faster alternative to the Whisper model without a significant loss in performance, especially for applications on resource-constrained hardware (paper.pdf, Page 1).

### Out-of-scope uses:
*   **Non-English Languages:** The model is distilled from English-only checkpoints (`medium.en` and `large-v2`) and trained on English datasets. It is not intended for speech recognition in languages other than English (paper.pdf, Page 6).
*   **Translation:** The model is designed for transcription (`<|transcribe|>`) and not for speech translation (`<|translate|>`) (generation_config.json).
*   **Timestamp Prediction:** The model was trained with timestamp probability set to 0.0, indicating it is not intended for predicting word-level timestamps (paper.pdf, Page 23, Table 15).

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
The model's performance is influenced by various factors related to acoustic conditions and speech characteristics. The development focused on maintaining robustness across these factors:
*   **Acoustic Conditions:** The model was evaluated against noisy conditions, including additive white noise and "pub noise" which simulates a busy restaurant environment (paper.pdf, Page 7).
*   **Audio Domains:** The model's robustness was tested across 10 distinct domains, including government interviews, audiobooks, podcasts, narrated Wikipedia, and telephone conversations (paper.pdf, Page 2, 6).
*   **Speaker Diversity:** The training data includes over 18,260 speakers to ensure generalization (paper.pdf, Page 5).
*   **Speaking Styles:** The data includes narrated, oratory, and spontaneous speech styles (paper.pdf, Page 21, Table 14).

### Evaluation factors:
The model was evaluated on a broad collection of speech recognition datasets to assess its ability to generalize across different domains and acoustic conditions. The evaluation factors include:
*   **In-Distribution (ID) vs. Out-of-Distribution (OOD) Data:** Performance was measured on both datasets seen during training (ID) and unseen datasets (OOD) to test generalization (paper.pdf, Page 7).
*   **Noise Levels:** Performance was measured on the LibriSpeech test-clean dataset with varying levels of additive white noise and pub noise, specified by the signal-to-noise ratio (SNR) (paper.pdf, Page 7).
*   **Audio Length:** The model was evaluated on both short-form (under 30 seconds) and long-form (over 10 minutes) audio files (paper.pdf, Page 7).

## Metrics
This section describes how the model's performance is evaluated.

### Model performance measures:
*   **Word Error Rate (WER):** This is the primary metric used to assess transcription accuracy. The text is normalized using the Whisper English normaliser before WER calculation to penalize only transcription errors, not formatting differences (paper.pdf, Page 23).
*   **Relative Latency:** Used to measure inference speed, computed as the ratio of the inference time of a given model to that of the Whisper `large-v2` checkpoint (paper.pdf, Page 8, Table 5).
*   **Real-Time Factor (RTF):** The ratio of the inference time to the audio duration, used for detailed latency measurements (paper.pdf, Page 29).
*   **Long-Form Evaluation Metrics:** To quantify hallucinations and repetitions, the following metrics were used (paper.pdf, Page 8):
    *   **5-gram word duplicates (5-Dup.):** The number of repeated 5-gram sequences.
    *   **Insertion Error Rate (IER):** The rate of words incorrectly inserted by the model.
    *   **Substitution Error Rate (SER):** The rate of words incorrectly substituted by the model.
    *   **Deletion Error Rate (DER):** The rate of words incorrectly deleted by the model.

### Decision thresholds:
A WER threshold (λ) was used during training to filter the pseudo-labelled data. This heuristic discards any training samples where the WER between the normalized ground truth and the normalized pseudo-label exceeds the threshold. A threshold of λ=10% was found to provide a good trade-off between data quality and quantity (paper.pdf, Page 4, 11).

### Variation approaches:
The model's performance was evaluated across a wide range of datasets to ensure robustness.
*   **Short-Form Evaluation:** Performance was measured on 15 short-form datasets, including 11 in-distribution (ID) and 4 out-of-distribution (OOD) datasets (paper.pdf, Page 7).
*   **Long-Form Evaluation:** Performance was measured on 4 OOD long-form datasets (paper.pdf, Page 8).
*   **Noise Robustness:** The model was evaluated on the LibriSpeech test-clean dataset with additive white noise and pub noise at various Signal-to-Noise Ratios (SNRs) (paper.pdf, Page 7).

### Datasets:
The model was evaluated on a total of 15 short-form and 4 long-form datasets.

**Out-of-Distribution (OOD) Short-Form Datasets (Total 224 hours):**
*   **CHIME-4:** 7 hours, 87 speakers. News broadcast domain (paper.pdf, Page 7, Table 4).
*   **Earnings-22:** 115 hours, unknown speakers. Financial meetings domain (paper.pdf, Page 7, Table 4).
*   **FLEURS:** 2 hours, 3 speakers. Narrated Wikipedia domain (paper.pdf, Page 7, Table 4).
*   **SPGISpeech:** 100 hours, unknown speakers. Financial meetings domain (paper.pdf, Page 7, Table 4).

**Out-of-Distribution (OOD) Long-Form Datasets (Total 171 hours):**
*   **Earnings-21:** 39 hours, unknown speakers. Financial meetings domain (paper.pdf, Page 7, Table 4).
*   **Earnings-22:** 115 hours, unknown speakers. Financial meetings domain (paper.pdf, Page 7, Table 4).
*   **Meanwhile:** 1 hour, 1 speaker. TV show domain (paper.pdf, Page 7, Table 4).
*   **Rev 16:** 16 hours, unknown speakers. Podcast domain (paper.pdf, Page 7, Table 4).

**In-Distribution (ID) Datasets:**
The model was also evaluated on the test splits of the 11 datasets used for training, including AMI, Common Voice 13, GigaSpeech, LibriSpeech, People's Speech, SwitchBoard, TED-LIUM, and VoxPopuli (paper.pdf, Page 7, 24).

### Motivation:
The evaluation datasets were chosen to cover a broad range of acoustic conditions, including different speakers, speaking styles, and domains. The use of out-of-distribution (OOD) datasets was specifically motivated by the need to measure the model's ability to generalize to unseen data and retain the robustness of the original Whisper model (paper.pdf, Page 7). The LibriSpeech dataset was used for noise robustness evaluation due to its high initial signal-to-noise ratio (SNR) (paper.pdf, Page 7).

### Preprocessing:
For all evaluations, the output transcriptions were standardized using the Whisper English normaliser, which handles punctuation, casing, and numerical expressions. This ensures that the WER metric primarily reflects transcription errors rather than formatting differences (paper.pdf, Page 23). For the short-form version of the Earnings-22 dataset, long audio files were segmented into shorter samples (up to 20 seconds) based on predicted timestamps for punctuation (paper.pdf, Page 22).

## Training Data
This section provides details about the datasets used to train the model.

### Datasets:
A large corpus of 21,170 hours of English speech data was assembled by combining nine publicly available datasets. The combined dataset features over 18,260 speakers and covers 10 distinct domains (paper.pdf, Page 5).

The datasets used are (paper.pdf, Page 6, Table 2; Page 20, Table 13):
*   **People's Speech:** 12,000 hours (Government, interviews)
*   **GigaSpeech:** 2,500 hours (Audiobook, podcast, YouTube)
*   **Common Voice 13:** 2,400 hours (Narrated Wikipedia)
*   **Fisher:** 1,960 hours (Telephone conversations)
*   **LibriSpeech:** 960 hours (Audiobooks)
*   **VoxPopuli:** 540 hours (European Parliament)
*   **TED-LIUM:** 450 hours (TED talks)
*   **SwitchBoard:** 260 hours (Telephone conversations)
*   **AMI:** 100 hours (Meetings)

### Motivation:
The datasets were chosen to create a large-scale, diverse training corpus, inspired by the SpeechStew methodology. This diversity in domains, speakers, and acoustic conditions is crucial for the knowledge distillation process to ensure the resulting smaller model retains the generalization ability and robustness of the large teacher model (paper.pdf, Page 2, 5).

### Preprocessing:
The training process used pseudo-labelling. The entire 21,170-hour training corpus was transcribed by the Whisper `large-v2` model using greedy decoding to generate "pseudo-labels" (paper.pdf, Page 5).

A filtering heuristic was then applied:
1.  Both the original ground-truth labels and the generated pseudo-labels were normalized using the Whisper English normaliser (paper.pdf, Page 4).
2.  The Word Error Rate (WER) was computed between the normalized ground-truth and the pseudo-label for each sample (paper.pdf, Page 4).
3.  Samples with a WER exceeding a certain threshold (tuned to 10%) were discarded from the training set to ensure only high-quality pseudo-labels were used (paper.pdf, Page 4, 11).

## Quantitative Analyses
This section presents disaggregated evaluation results.

### Unitary results:
The model's performance was evaluated across multiple datasets and conditions.

**Performance on OOD Short-Form Datasets (Average WER):**
*   `distil-large-v2`: 10.1%
*   `large-v2` (teacher): 9.1%
*   `distil-medium.en`: 11.1%
(paper.pdf, Page 8, Table 5)

**Performance on OOD Long-Form Datasets (Average WER):**
*   `distil-large-v2`: 11.6%
*   `large-v2` (teacher): 11.7%
*   `distil-medium.en`: 12.4%
(paper.pdf, Page 8, Table 5)

**Per-Dataset WER Scores:**
Detailed WER scores for `distil-large-v2` on 15 short-form and 5 long-form datasets are available in the paper (paper.pdf, Page 24, Tables 16 & 17). For example, on the OOD SPGISpeech dataset, `distil-large-v2` achieves a WER of 3.3%, outperforming the `large-v2` teacher model's 3.8% (paper.pdf, Page 24, Table 16).

**Robustness to Noise:**
Performance degrades as noise increases. On the LibriSpeech test-clean set with additive pub noise, the WER of `distil-large-v2` degrades less severely than LibriSpeech-trained models and approaches the performance of the Whisper teacher model, especially below 10 dB SNR (paper.pdf, Page 9, Figure 2).

**Hallucination and Repetition Analysis (Long-Form):**
Averaged over four OOD long-form datasets, `distil-large-v2` showed improved robustness to hallucinations compared to its teacher:
*   **Insertion Error Rate (IER):** 2.1% (vs. 3.3% for `large-v2`)
*   **Repeated 5-gram Duplicates (5-Dup.):** 18,503 (vs. 23,792 for `large-v2`)
However, it had a slightly higher Substitution Error Rate (SER) of 5.3% (vs. 3.9% for `large-v2`) (paper.pdf, Page 11, Table 8).

## Memory or Hardware Requirements
This section outlines the memory or hardware requirements for loading, deploying, and training the model.

### Loading Requirements:
The `distil-large-v2` model has 756 million parameters, which is a 51% reduction compared to the 1550 million parameters of the `large-v2` teacher model (paper.pdf, Page 1, 7). Latency benchmarks were performed on a single A100 40GB GPU and a T4 16GB GPU, suggesting these are suitable for running the model (paper.pdf, Page 23).

### Deploying Requirements:
The model is 5.8 times faster than the `large-v2` model, making it more suitable for resource-constrained environments (paper.pdf, Page 1). When used as an assistant for speculative decoding with `large-v2`, the total parameter count increases by only 8% because the encoder can be shared between the two models (paper.pdf, Page 10).

### Training or Fine-tuning Requirements:
The model was trained using data parallelism across TPU v4-8 accelerators with bfloat16 precision (paper.pdf, Page 22).

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
*   **Performance Trade-off:** While significantly faster, `distil-large-v2` performs approximately 1% absolute WER worse than the `large-v2` teacher model on average across OOD short-form datasets (paper.pdf, Page 8).
*   **Substitution Errors:** The distillation process leads to a higher substitution error rate (SER) compared to the teacher model, although this is offset by a lower insertion error rate (IER), resulting in better overall long-form WER (paper.pdf, Page 10).
*   **Single Speaker/Uncommon Words:** The model's performance can degrade on audio that contains a single speaker with a high frequency of uncommon words, as seen in the "Meanwhile" dataset evaluation (paper.pdf, Page 9).
*   **Repetition Errors:** While improved over the teacher model, Distil-Whisper is still prone to repetition errors, a known issue for Seq2Seq ASR models that requires further research (paper.pdf, Page 10).

### Recommendations:
*   **Use with Speculative Decoding:** For the best latency improvement without sacrificing any accuracy, it is recommended to pair Distil-Whisper with the original Whisper model for speculative decoding. This provides a 2x speed-up while mathematically guaranteeing the same outputs as the larger model (paper.pdf, Page 1).
*   **Use with Flash Attention 2:** For batch inference on modern GPUs, using Flash Attention 2 is highly recommended as it significantly improves inference time, especially for Distil-Whisper due to its deep encoder architecture (paper.pdf, Page 29).
*   **Chunk Length for Long-Form Audio:** The optimal `chunk_length` for the chunked long-form transcription algorithm should be chosen based on the audio distribution. For Distil-Whisper, a chunk length of 15 seconds performed best on the TED-LIUM validation set, in contrast to 30 seconds for the original Whisper model (paper.pdf, Page 28).
