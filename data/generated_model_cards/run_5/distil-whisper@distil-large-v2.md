## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by Sanchit Gandhi, Patrick von Platen, and Alexander M. Rush from Hugging Face (Distil-Whisper Paper, page 1).

### Model date:
The paper describing the model was submitted on November 1, 2023 (Distil-Whisper Paper, page 1).

### Model version:
This model is `distil-large-v2`, a distilled version of the `large-v2` Whisper model (Distil-Whisper Paper, page 6). It differs from the original `large-v2` model in the following ways:
*   **Size and Speed**: It has 51% fewer parameters (756M vs 1550M) and is 5.8 times faster than the `large-v2` model (Distil-Whisper Paper, page 1; page 7, Table 3; page 8, Table 5).
*   **Architecture**: It uses the same 32-layer encoder as `large-v2` but has a decoder with only 2 layers, initialized from the first and last decoder layers of the teacher model (Distil-Whisper Paper, page 6, Figure 1).
*   **Performance**: It performs to within 1% Word Error Rate (WER) of the original `large-v2` model on out-of-distribution short-form audio (Distil-Whisper Paper, page 1). On long-form audio, it outperforms the original model by 0.1% WER, being less prone to hallucinations (Distil-Whisper Paper, page 8).

### Model type:
*   **General Type**: The model is a sequence-to-sequence (Seq2Seq) Transformer for Automatic Speech Recognition (ASR) (Distil-Whisper Paper, page 3).
*   **Architecture**: It is a "Whisper" type model, specifically a `WhisperForConditionalGeneration` architecture. It consists of a transformer-based encoder and decoder (config.json, "model_type", "architectures"; Distil-Whisper Paper, page 3).
    *   **Encoder**: 32 encoder layers, 20 attention heads, a model dimension of 1280, and a feed-forward network dimension of 5120 (config.json, "encoder_layers", "encoder_attention_heads", "d_model", "encoder_ffn_dim"). The encoder is copied from the `large-v2` teacher model and frozen during training (Distil-Whisper Paper, page 6).
    *   **Decoder**: 2 decoder layers, 20 attention heads, a model dimension of 1280, and a feed-forward network dimension of 5120 (config.json, "decoder_layers", "decoder_attention_heads", "d_model", "decoder_ffn_dim").
*   **Size**: The model has 756 million parameters (Distil-Whisper Paper, page 7, Table 3).
*   **Context Length**: The model has a fixed receptive field corresponding to 30-second audio inputs (480,000 samples) (preprocessor_config.json, "n_samples"; Distil-Whisper Paper, page 5). The maximum source position length is 1500 tokens, and the maximum target position length is 448 tokens (config.json, "max_source_positions", "max_target_positions").
*   **Vocabulary**: The model uses a vocabulary of 51,865 tokens, which includes special tokens for multilingual transcription and tasks like translation (config.json, "vocab_size"; tokenizer.json).

### Training details:
The model was trained using knowledge distillation from the Whisper `large-v2` model (Distil-Whisper Paper, page 1).

*   **Algorithm**: The training objective is a weighted sum of two knowledge distillation terms:
    1.  **Pseudo-Labelling (PL)**: A "sequence-level" distillation where the student model is trained on transcriptions generated by the teacher model (pseudo-labels) using a standard cross-entropy loss (Distil-Whisper Paper, page 4).
    2.  **Kullback-Leibler (KL) Divergence**: A "word-level" distillation where the student model is trained to match the full probability distribution of the teacher model for the next token at each position (Distil-Whisper Paper, page 4).
*   **Training Objective**: The final objective is `L_KD = 0.8 * L_KL + 1.0 * L_PL` (Distil-Whisper Paper, page 4).
*   **Pseudo-Label Filtering**: To ensure high-quality training data, a filtering heuristic was used. Pseudo-labels were generated for the training data, and any sample where the Word Error Rate (WER) between the normalized ground truth and the normalized pseudo-label was greater than 10% was discarded (Distil-Whisper Paper, page 4, 11).
*   **Initialization**: The student model's 32-layer encoder was initialized by copying the weights from the `large-v2` teacher model and was frozen during training. The 2-layer decoder was initialized by copying weights from the 1st and 32nd layers of the teacher's decoder (Distil-Whisper Paper, page 4, 6).
*   **Hyperparameters**:
    *   **Device**: TPU v4-8 (Distil-Whisper Paper, page 22).
    *   **Updates**: 80,000 optimization steps (Distil-Whisper Paper, page 6).
    *   **Batch Size**: 256 (Distil-Whisper Paper, page 6).
    *   **Precision**: bfloat16 (Distil-Whisper Paper, page 22).
    *   **Optimizer**: AdamW with β1=0.9, β2=0.999, and weight decay of 10e-8 (Distil-Whisper Paper, page 23, Table 15).
    *   **Learning Rate Schedule**: Slanted triangular learning rate (STLR), warming up from 0 to a maximum of 1e-4 over 500 steps, then linearly decaying to 0 (Distil-Whisper Paper, page 23).
    *   **KL Softmax Temperature**: 2.0 (Distil-Whisper Paper, page 22).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The model is intended for Automatic Speech Recognition (ASR) for the English language (Distil-Whisper Paper, page 1; generation_config.json, "language"). It is specifically designed as a smaller, faster alternative to the large Whisper models, making it suitable for low-latency or resource-constrained environments (Distil-Whisper Paper, page 1).

A key use case is as an "assistant model" for speculative decoding with the full Whisper `large-v2` model. This setup provides a 2x speed-up in inference while mathematically guaranteeing the same outputs as the original `large-v2` model, making it a drop-in replacement for existing Whisper pipelines (Distil-Whisper Paper, page 2, 11).

The model is robust to various acoustic conditions and domains and is effective for both short-form and long-form transcription (Distil-Whisper Paper, page 2, 5).

### Primary intended users:
The primary users are developers and researchers in the field of speech recognition who require a robust ASR model that is faster and less resource-intensive than the original Whisper models (Distil-Whisper Paper, page 1, 2).

### Out-of-scope uses:
The model is designed for English speech-to-text transcription and is not intended for:
*   Transcription of languages other than English.
*   Any task other than speech recognition (e.g., text generation without an audio prompt, text-to-speech).
*   Use in high-stakes applications where transcription errors could have severe consequences, without further testing and fine-tuning. Although it is less prone to hallucinations than the original Whisper model, it can still produce transcription errors, including substitutions (Distil-Whisper Paper, page 8).

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
The model's performance is influenced by a variety of factors related to the audio input. The training data was specifically selected to cover a wide range of these factors to ensure robustness (Distil-Whisper Paper, page 5, 7).
*   **Acoustic Conditions**: The model's robustness to noisy audio conditions was a key consideration (Distil-Whisper Paper, page 2). This includes background noise and different signal-to-noise ratios (SNR) (Distil-Whisper Paper, page 7).
*   **Audio Domains**: The model was trained and evaluated on data from over 10 distinct domains, including government interviews, audiobooks, podcasts, narrated Wikipedia, telephone conversations, and financial meetings (Distil-Whisper Paper, page 5, 6).
*   **Speaker Diversity**: The training data includes over 18,260 speakers with diverse accents and speaking styles (Distil-Whisper Paper, page 5, 20).
*   **Speaking Style**: The data includes narrated, oratory, and spontaneous speech (Distil-Whisper Paper, page 21, Table 14).

### Evaluation factors:
The model was evaluated against the factors listed above to measure its generalization and robustness capabilities. Specifically, evaluations were conducted on:
*   **In-Distribution (ID) vs. Out-of-Distribution (OOD) Data**: Performance was measured on test sets corresponding to the training data (ID) and on entirely unseen datasets (OOD) to assess generalization (Distil-Whisper Paper, page 7).
*   **Noise Levels**: Performance was benchmarked on the LibriSpeech test-clean dataset with varying amounts of additive white noise and pub noise to measure noise robustness (Distil-Whisper Paper, page 7).
*   **Audio Length**: The model was evaluated on both short-form (under 30 seconds) and long-form (several minutes to hours) audio transcription tasks (Distil-Whisper Paper, page 7).

## Metrics
This section describes how the model's performance is evaluated.

### Model performance measures:
The primary metric used to assess transcription accuracy is the **Word Error Rate (WER)** (Distil-Whisper Paper, page 1).

For long-form evaluation and analyzing hallucinations, the following additional metrics were used (Distil-Whisper Paper, page 8):
*   **Insertion Error Rate (IER)**: To quantify passages of text predicted but not spoken.
*   **Substitution Error Rate (SER)**: To quantify the frequency of word substitutions.
*   **Deletion Error Rate (DER)**: To quantify the frequency of word deletions.
*   **Repeated 5-gram word duplicates (5-Dup.)**: To measure repetitive generation errors.

Inference speed is measured by **relative latency** compared to the `large-v2` checkpoint and by the **real-time factor (RTF)** (Distil-Whisper Paper, page 8, Table 5; page 29, Table 24).

### Decision thresholds:
A **WER threshold (λ)** was used during the data preparation phase for training. To create a high-quality training set from pseudo-labels, any audio-text pair where the WER between the teacher model's transcription and the ground-truth transcription exceeded a certain threshold was discarded. A threshold of λ = 10% was found to provide a good trade-off between data quality and quantity (Distil-Whisper Paper, page 4, 11).

### Variation approaches:
Model performance was measured across a wide range of conditions to ensure robustness. This included:
*   **Cross-Domain Evaluation**: The model was evaluated on 15 short-form and 4 long-form datasets spanning different domains (e.g., financial, podcasts, news) (Distil-Whisper Paper, page 7).
*   **In-Distribution vs. Out-of-Distribution**: Performance was compared on datasets seen during training (ID) versus entirely new datasets (OOD) to measure generalization (Distil-Whisper Paper, page 7).
*   **Noise Robustness**: The model was evaluated against varying levels of additive white noise and pub noise, measured in signal-to-noise ratio (SNR), to assess its performance under different acoustic conditions (Distil-Whisper Paper, page 7, 10).

## Evaluation Data
This section provides details about the datasets used to evaluate the model.

### Datasets:
The model was evaluated on a total of 15 short-form and 5 long-form datasets.

**Short-Form Datasets (Out-of-Distribution):**
*   **CHIME-4**: 7 hours, 87 speakers, news broadcast domain (Distil-Whisper Paper, page 7, Table 4).
*   **Earnings-22**: 115 hours, unknown speakers, financial meetings domain (Distil-Whisper Paper, page 7, Table 4).
*   **FLEURS**: 2 hours, 3 speakers, narrated Wikipedia domain (Distil-Whisper Paper, page 7, Table 4).
*   **SPGISpeech**: 100 hours, unknown speakers, financial meetings domain (Distil-Whisper Paper, page 7, Table 4).

**Long-Form Datasets (Out-of-Distribution):**
*   **Earnings-21**: 39-hour corpus of company earnings calls (Distil-Whisper Paper, page 22).
*   **Earnings-22**: 119-hour test set of earnings calls (Distil-Whisper Paper, page 22).
*   **Meanwhile**: A collection of 64 segments from "The Late Show with Stephen Colbert" (Distil-Whisper Paper, page 22).
*   **Rev 16**: A set of 16 podcast recordings used to benchmark ASR systems (Distil-Whisper Paper, page 22).

**In-Distribution Datasets:**
The model was also evaluated on the test splits of the 11 datasets used for training, including AMI, Call Home, Common Voice 13, GigaSpeech, LibriSpeech, People's Speech, SwitchBoard, TED-LIUM, and VoxPopuli (Distil-Whisper Paper, page 7, 24).

### Motivation:
The evaluation datasets were chosen to be broad and diverse, covering multiple acoustic conditions, domains, speakers, and speaking styles. The inclusion of four out-of-distribution (OOD) datasets for both short-form and long-form evaluation was specifically to measure the model's ability to generalize to unseen data, which is a key indicator of its robustness (Distil-Whisper Paper, page 7).

### Preprocessing:
For evaluation, the generated transcriptions are normalized using the Whisper English normaliser. This process standardizes text by removing or converting specific symbols, managing whitespace, and correcting spellings to ensure that the WER metric primarily reflects transcription errors rather than formatting differences (Distil-Whisper Paper, page 23). The audio inputs are processed into 80-dimensional log-Mel spectrograms with a sampling rate of 16000 Hz (preprocessor_config.json, "feature_size", "sampling_rate").

## Training Data
This section provides details about the datasets used to train the model.

### Datasets:
The model was trained on a large-scale corpus assembled from nine publicly available speech recognition datasets, totaling 21,770 hours of audio from over 18,260 speakers across 10 domains (Distil-Whisper Paper, page 5). The component datasets are:
*   **People's Speech**: 12,000 hours (Government, interviews)
*   **GigaSpeech**: 2,500 hours (Audiobook, podcast, YouTube)
*   **Common Voice 13**: 2,400 hours (Narrated Wikipedia)
*   **Fisher**: 1,960 hours (Telephone conversations)
*   **LibriSpeech**: 960 hours (Audiobooks)
*   **VoxPopuli**: 540 hours (European Parliament)
*   **TED-LIUM**: 450 hours (TED talks)
*   **SwitchBoard**: 260 hours (Telephone conversations)
*   **AMI**: 100 hours (Meetings)
(Distil-Whisper Paper, page 6, Table 2).

### Motivation:
The datasets were chosen to create a large and diverse corpus, inspired by the SpeechStew methodology (Chan et al., 2021). This diversity in speakers, domains, and acoustic conditions is crucial for the knowledge distillation process to produce a student model that retains the robustness of the original large-scale Whisper model (Distil-Whisper Paper, page 5, 7).

### Preprocessing:
The training process involved several preprocessing steps:
1.  **Pseudo-Labelling**: Transcriptions for the entire 21,770-hour dataset were generated using the Whisper `large-v2` model with greedy decoding. These transcriptions served as the training targets (pseudo-labels) (Distil-Whisper Paper, page 5).
2.  **Text Normalization**: Both the ground-truth labels and the generated pseudo-labels were normalized using the Whisper English normaliser (Distil-Whisper Paper, page 4). This standardizes text to allow for accurate WER calculation.
3.  **WER Filtering**: A filtering heuristic was applied to select only high-quality pseudo-labels. Any training sample where the WER between the normalized ground-truth and the normalized pseudo-label exceeded 10% was discarded. This filtering removed 45.4% of the initial data (Distil-Whisper Paper, page 4, 12).
4.  **Audio Processing**: Audio files were loaded and resampled to 16,000 Hz. They were then converted into 80-dimensional log-Mel spectrograms, which serve as the input to the model's encoder (preprocessor_config.json).

## Memory or Hardware Requirements
This section outlines the memory or hardware requirements for loading, deploying, and training the model.

### Loading Requirements:
The model has 756 million parameters (Distil-Whisper Paper, page 7, Table 3). For inference in float16 precision, it would require approximately 1.5 GB of GPU VRAM.

### Deploying Requirements:
Inference speed benchmarks were provided for two types of GPUs:

*   **NVIDIA A100 (40GB)**:
    *   With base attention, the real-time factor (RTF) is 25.1 x 10⁻³ at batch size 1 and 6.5 x 10⁻³ at batch size 16 (Distil-Whisper Paper, page 30, Table 24).
    *   With Flash Attention 2, the RTF is 24.2 x 10⁻³ at batch size 1 and 4.4 x 10⁻³ at batch size 16 (Distil-Whisper Paper, page 30, Table 24).
*   **NVIDIA T4 (16GB)**:
    *   With base attention, the RTF is 38.0 x 10⁻³ at batch size 1 and 31.2 x 10⁻³ at batch size 16 (Distil-Whisper Paper, page 30, Table 25).
    *   With Flash Attention, the RTF is 27.4 x 10⁻³ at batch size 1 and 20.1 x 10⁻³ at batch size 16 (Distil-Whisper Paper, page 30, Table 25).

### Training or Fine-tuning Requirements:
The model was trained using data parallelism across TPU v4-8 accelerators with bfloat16 precision and gradient checkpointing (Distil-Whisper Paper, page 22).

## Ethical Considerations
This section discusses the ethical considerations in model development, including challenges, risks, and solutions. 

The provided documentation does not contain a specific section on ethical considerations. However, based on the model's function as a speech recognition system, potential risks can be inferred:

*   **Transcription Errors**: The model, like any ASR system, is not perfect and can make errors. Mis-transcribing names, technical terms, or sensitive content could have negative consequences. The model has a higher substitution error rate than its teacher model (Distil-Whisper Paper, page 8).
*   **Bias**: The model's performance may vary across different accents, dialects, and demographic groups. Although trained on a diverse dataset, biases present in the source data could be inherited and amplified by the model. The paper does not include a disaggregated analysis of performance across demographic groups.
*   **Surveillance**: ASR technology can be used for surveillance purposes. As a smaller and faster model, Distil-Whisper could potentially lower the barrier for such applications.
*   **Hallucinations**: While the paper notes that Distil-Whisper is less prone to hallucinations (generating text not present in the audio) than the original Whisper model, this issue is not entirely eliminated and remains a risk, especially in long-form audio with silent segments (Distil-Whisper Paper, page 8).

The developers have made their training code, inference code, and models publicly accessible to facilitate further research (Distil-Whisper Paper, page 1).

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
*   **Substitution Errors**: The distilled model is more prone to substitution errors than the original `large-v2` model, with a Substitution Error Rate (SER) that is 1.4% higher on average for long-form audio (Distil-Whisper Paper, page 8).
*   **Performance on Niche Vocabulary**: The model's performance can degrade on audio that contains a high frequency of uncommon words. For example, it performed worse than `large-v2` on the "Meanwhile" dataset, which features a single speaker with such vocabulary (Distil-Whisper Paper, page 9).
*   **Repetition Errors**: While improved compared to the teacher model, Distil-Whisper is still more prone to repetition errors in long-form transcription than encoder-only CTC-based models like wav2vec 2.0 (Distil-Whisper Paper, page 8).
*   **Encoder Depth**: The model's strong performance relies heavily on the deep 32-layer encoder inherited from the teacher model. Reducing the encoder size (e.g., to 16 layers) leads to a significant degradation in OOD WER performance (3.1% absolute increase) (Distil-Whisper Paper, page 12).

### Recommendations:
*   **Speculative Decoding**: For applications requiring the exact output of the Whisper `large-v2` model but with lower latency, it is recommended to use `distil-large-v2` as an assistant model for speculative decoding. This provides a 2x inference speed-up with only an 8% increase in total parameters (Distil-Whisper Paper, page 2, 11).
*   **Chunk Length for Long-Form Transcription**: When using the chunked algorithm for long-form transcription, the optimal chunk length should be chosen carefully. For Distil-Whisper, a chunk length of 15 seconds yielded the best performance on the TED-LIUM validation set, suggesting that the chunk length should be tuned based on the characteristics of the target audio data (Distil-Whisper Paper, page 29).
*   **Use with Flash Attention**: For optimal inference speed, especially at higher batch sizes, it is recommended to use the model with Flash Attention 2, as it provides a significant performance boost for the Distil-Whisper architecture (Distil-Whisper Paper, page 29).
