## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by James Betker (setup.py; paper/paper.pdf, p. 1). The project is open-sourced on GitHub at https://github.com/neonbjb/tortoise-tts (setup.py; paper/paper.pdf, p. 1). The author's email is james@adamant.ai (setup.py).

### Model date:
The models were trained over a period of one year on a cluster of 8 NVIDIA RTX-3090s (paper/paper.pdf, p. 4). The associated research paper was submitted to arXiv on May 23, 2023 (paper/paper.pdf, p. 1). The version specified in the setup file is 2.1.3 (setup.py).

### Model version:
The model is referred to as TorToise-v2 (paper/paper.pdf, p. 3). The provided software package version is 2.1.3 (setup.py). This version is a text-to-speech system that combines an autoregressive model with a diffusion decoder to generate high-quality, multi-voice speech (paper/paper.pdf, p. 1, 3).

### Model type:
TorToise is a multi-voice text-to-speech (TTS) system that applies advances from image generation to speech synthesis (paper/paper.pdf, p. 1). It is composed of several distinct neural networks that work in sequence (paper/paper.pdf, p. 3, Figure 1).

The main components are:
1.  **Autoregressive (AR) Model**: This model uses a GPT-2 architecture to convert a sequence of text tokens into a sequence of speech tokens, conditioned on a speech conditioning input (paper/paper.pdf, p. 3, 8).
    *   **Architecture**: It is a 30-layer Transformer with a model dimension of 1024 and 16 attention heads (tortoise/models/autoregressive.py; paper/paper.pdf, p. 9). It supports a maximum of 402 text tokens and 604 MEL tokens (tortoise/models/autoregressive.py). It uses learned positional embeddings for both text and MEL tokens (paper/paper.pdf, p. 8).

2.  **CLVP (Contrastive Language-Voice Pretrained) Model**: Similar to CLIP, this model is used to rank the output candidates from the autoregressive model to select the best ones for further processing (paper/paper.pdf, p. 4, 10).
    *   **Architecture**: It consists of two Transformer stacks. The text encoder has a depth of 12 layers, a dimension of 512, and 8 heads. The speech encoder has a depth of 12 layers, a dimension of 512, and 8 heads (tortoise/models/clvp.py; paper/paper.pdf, p. 10). It supports a maximum text length of 350 tokens and a speech token length of 430 (tortoise/models/clvp.py).

3.  **CVVP (Contrastive Voice-Voice Pretrained) Model**: This model is also used in conjunction with CLVP to filter the outputs from the autoregressive model, comparing the generated speech to the reference voice clips (api.py; tortoise/models/cvvp.py).
    *   **Architecture**: It uses a `CollapsingTransformer` architecture with a model dimension of 512 and 8 transformer heads (tortoise/models/cvvp.py).

4.  **Diffusion Decoder (DDPM)**: This model converts the speech tokens (or latent vectors from the AR model) into a high-quality MEL spectrogram (paper/paper.pdf, p. 3, 11).
    *   **Architecture**: It is a bespoke architecture combining residual convolutions with 10 layers of dense self-attention, resembling a U-Net without upsampling or downsampling (paper/paper.pdf, p. 11). It has a model dimension of 1024 and 16 attention heads (tortoise/models/diffusion_decoder.py; paper/paper.pdf, p. 12).

5.  **Vocoder**: A pre-trained UnivNet model is used as the final stage to convert the generated MEL spectrograms into audio waveforms (paper/paper.pdf, p. 1; tortoise/models/vocoder.py; api.py).

6.  **Classifier**: An `AudioMiniEncoderWithClassifierHead` is included to determine the probability that a given audio clip was generated by Tortoise (api.py; tortoise/models/classifier.py).

### Training details:
The training process involves several stages for the different components of the model (paper/paper.pdf, p. 3).

*   **Autoregressive Model Training**:
    *   The AR model is a GPT-2 architecture trained to predict speech tokens conditioned on text (paper/paper.pdf, p. 8).
    *   **Hyperparameters**: It was trained with a batch size of 1024 for 119 million samples. The learning rate was 1e-4 with a 500-step warmup. The optimizer used was Adam with B1=0.9 and B2=0.96 (paper/paper.pdf, p. 9, Table 2).
    *   **Tokenization**: A custom BPE tokenizer with a vocabulary of 256 tokens is used for text (paper/paper.pdf, p. 9, Table 2; tortoise/data/tokenizer.json).

*   **Diffusion Model Training**:
    *   The diffusion model is trained to convert discrete speech codes into MEL spectrograms. It is then fine-tuned on the latent space from the AR model's outputs, a process called the "TorToise Trick" (paper/paper.pdf, p. 4).
    *   **Algorithm**: It uses a Denoising Diffusion Probabilistic Model (DDPM) framework (paper/paper.pdf, p. 2).
    *   **Hyperparameters**: It was trained with a batch size of 512 for 65 million samples. The learning rate was 1e-5 with a 1000-step warmup (paper/paper.pdf, p. 12, Table 4).
    *   **Conditioning**: During training, classifier-free guidance is used, where 15% of the time, the speech conditioning signal and AR model activations are dropped and replaced with a learned embedding (paper/paper.pdf, p. 11).

*   **CLVP Model Training**:
    *   CLVP is trained on pairs of audio clips and text in a contrastive setting to act as a discriminator for re-ranking AR model outputs (paper/paper.pdf, p. 4).
    *   **Hyperparameters**: It was trained with a batch size of 1024 for 80 million samples, using a contrastive loss. The learning rate was 3e-4 with a 500-step warmup (paper/paper.pdf, p. 10, Table 3).

*   **Inference Sampling**:
    *   **Autoregressive**: Nucleus sampling is used with P=0.8, a repetition penalty of 2.0, and a softmax temperature of 0.8 (paper/paper.pdf, p. 4; api.py).
    *   **Diffusion**: DDIM sampling is used with a linear schedule, 64 sampling steps, and a conditioning-free guidance constant of 2 (paper/paper.pdf, p. 5; tortoise/utils/diffusion.py).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The primary intended use of TorToise is as an expressive, multi-voice text-to-speech (TTS) system (paper/paper.pdf, p. 1). Its key capabilities include:

*   **High-Quality Speech Generation**: Generating high-quality, realistic speech from text (paper/paper.pdf, p. 3).
*   **Voice Cloning**: Mimicking a specific voice by providing one or more reference audio clips (approximately 10 seconds each) of the target speaker. This is referred to as the "speech conditioning input" (api.py; paper/paper.pdf, p. 3).
*   **Voice Combination**: Creating a new voice with traits from multiple parent voices by combining their reference clips (tortoise-tts.ipynb).
*   **Random Voice Generation**: Generating speech in a random voice without any reference clips (tortoise-tts.ipynb).
*   **Long-form Reading**: The model can be used to read long text files by splitting the text into smaller segments and synthesizing them sequentially (do_longform_tts.py).

The input-output structure is as follows:
*   **Input**: A string of text and optionally one or more reference audio clips (as PyTorch tensors of 22.05kHz waveform data) or pre-computed conditioning latents (api.py).
*   **Output**: A PyTorch tensor containing the generated audio waveform at a sample rate of 24kHz (api.py).

### Primary intended users:
The model is intended for researchers, developers, and practitioners in the field of speech synthesis and machine learning who are interested in high-quality, multi-voice TTS (setup.py; tortoise-tts.ipynb).

### Out-of-scope uses:
The provided repository does not explicitly list out-of-scope uses. However, given the model's voice cloning capabilities, any use for malicious purposes such as creating deepfakes, impersonation for fraudulent activities, or generating hate speech would be considered misuse. The inclusion of a classifier to detect generated audio suggests an awareness of such potential misuse (classify.py).

## How to Use
This section outlines how to use the model.

To use the model, you first need to instantiate the `TextToSpeech` class. This will download the necessary model weights (tortoise-tts.ipynb).

```python
from tortoise.api import TextToSpeech
from tortoise.utils.audio import load_voice, load_voices
import torchaudio
import IPython

# This will download all the models used by Tortoise from the HF hub.
tts = TextToSpeech()
```
(Source: tortoise-tts.ipynb)

**Generating Speech with a Preset Voice**

You can generate speech using one of the pre-packaged voices.

```python
# The text to be spoken.
text = "The expressiveness of autoregressive transformers is literally nuts! I absolutely adore them."

# Pick a voice from the 'tortoise/voices' directory
voice = 'tom'

# Load the voice samples and conditioning latents.
voice_samples, conditioning_latents = load_voice(voice)

# Generate speech with a preset.
# Options: {"ultra_fast", "fast", "standard", "high_quality"}
preset = "fast"
gen = tts.tts_with_preset(text, voice_samples=voice_samples, conditioning_latents=conditioning_latents, 
                          preset=preset)

# Save and play the generated audio.
torchaudio.save('generated.wav', gen.squeeze(0).cpu(), 24000)
IPython.display.Audio('generated.wav')
```
(Source: tortoise-tts.ipynb; tortoise_tts.py)

**Generating Speech with a Random Voice**

The model can also generate speech using a random voice.

```python
gen = tts.tts_with_preset(text, voice_samples=None, conditioning_latents=None, preset=preset)
torchaudio.save('generated_random.wav', gen.squeeze(0).cpu(), 24000)
IPython.display.Audio('generated_random.wav')
```
(Source: tortoise-tts.ipynb)

**Combining Voices**

You can combine multiple voices to create a new one.

```python
# Combine voices 'pat' and 'william'
voice_samples, conditioning_latents = load_voices(['pat', 'william'])

gen = tts.tts_with_preset("They used to say that if man was meant to fly, he’d have wings. But he did fly. He discovered he had to.", 
                          voice_samples=voice_samples, conditioning_latents=conditioning_latents, preset=preset)
torchaudio.save('captain_kirkard.wav', gen.squeeze(0).cpu(), 24000)
IPython.display.Audio('captain_kirkard.wav')
```
(Source: tortoise-tts.ipynb)

**Example Outputs**

You can listen to various handpicked audio samples generated by the model for different voices and texts at the project's example page (examples/result.html). This includes comparisons with other models like Tacotron2 and long-form speech generation (examples/result.html).

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
*   **Reference Audio Quality**: The quality of the `voice_samples` provided for voice cloning significantly impacts the output. The training data pipeline for the extended dataset included classifiers to remove audio with background noise, music, poor quality, multiple voices, and reverb, indicating these are critical factors (paper/paper.pdf, p. 6, Appendix A).
*   **Prosody and Tone of Reference Audio**: The model infers vocal characteristics like tone and prosody from the conditioning input (reference audio) to guide the speech generation (paper/paper.pdf, p. 4).
*   **Voice Diversity vs. Intelligibility**: The `clvp_cvvp_slider` (or `voice_diversity_intelligibility_slider`) parameter balances how closely the generated speech follows the text versus how similar it sounds to the reference voice. Values closer to 1 prioritize text adherence, while values closer to 0 prioritize voice similarity (api.py; tortoise_tts.py).
*   **Generation Parameters**: Hyperparameters such as `temperature`, `top_p`, `repetition_penalty`, and `diffusion_iterations` directly influence the quality, diversity, and realism of the generated audio (api.py).

### Evaluation factors:
*   **Generation Hyperparameters**: The `param_sweep.py` script performs a sweep over hyperparameters like `top_p`, `temperature`, `diffusion_temperature`, and `cond_free_k`, indicating these are key factors analyzed during evaluation (scripts/param_sweep.py).
*   **Intelligibility**: An open-source wav2vec model is used to characterize the "intelligibility" of a speech segment during evaluation (paper/paper.pdf, p. 5).
*   **Sample Similarity**: The CLVP model is used to produce a distance metric between real and generated samples, similar to the FID score used in image generation, to evaluate sample quality (paper/paper.pdf, p. 5).

## Metrics
This section describes how the model's performance is evaluated.

### Model performance measures:
*   **Training Losses**: The different components of the model were trained using standard loss functions:
    *   **VQVAE & Diffusion Decoder**: Mean Squared Error (MSE) loss (paper/paper.pdf, p. 7, 11).
    *   **Autoregressive Model & CLVP**: Cross-Entropy loss (paper/paper.pdf, p. 9, 10).
*   **Evaluation Metrics**:
    *   **CLVP Distance**: A distance metric produced by the CLVP model is used to compare real and generated samples, analogous to the FID score in image generation (paper/paper.pdf, p. 5).
    *   **Intelligibility Score**: A wav2vec model is used to measure the intelligibility of the generated speech (paper/paper.pdf, p. 5).

### Decision thresholds:
*   **Nucleus Sampling (Top-p)**: During autoregressive decoding, nucleus sampling is used with a `top_p` value of 0.8. This means that the model samples from the smallest set of tokens whose cumulative probability is greater than 0.8 (paper/paper.pdf, p. 4; api.py).

## Training Data
This section provides details about the datasets used to train the model.

### Datasets:
The model was trained on a large and diverse collection of speech data:
1.  **LibriTTS**: A corpus derived from LibriSpeech, containing transcribed speech (paper/paper.pdf, p. 5).
2.  **HiFiTTS**: A multi-speaker English TTS dataset (paper/paper.pdf, p. 5).
    *   Together, LibriTTS and HiFiTTS contain 896 hours of transcribed speech (paper/paper.pdf, p. 5).
3.  **Extended Dataset**: An independently built dataset of 49,000 hours of cleaned audio clips from audiobooks and podcasts scraped from the web (paper/paper.pdf, p. 5, Appendix A).

### Motivation:
The goal was to train a large language model for speech, which required a vast amount of data. The combination of existing high-quality datasets and a large-scale scraped dataset was chosen to meet this requirement (paper/paper.pdf, p. 5).

### Preprocessing:
The extended dataset underwent a significant preprocessing pipeline:
*   Audio was split on 500ms silences, and clips between 5-20 seconds were retained.
*   A series of trained classifiers were used to filter out clips containing:
    *   Background noise
    *   Music
    *   Poor quality (e.g., phone calls)
    *   Multiple voices speaking at once
    *   Reverb
*   The cleaned audio clips were transcribed using a wav2vec2-large model that was fine-tuned to predict punctuation (commas, quotation marks, etc.) (paper/paper.pdf, p. 6, Appendix A).

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
*   **Performance Speed**: The model is computationally intensive and slow, which is why it is named "TorToise". Inference for a single sentence can take up to a minute even on a GPU (tortoise-tts.ipynb).
*   **Output Artifacts**: The autoregressive model can sometimes produce a harsh-sounding artifact at the end of speech. A workaround is implemented in the code to mitigate this, but it is specific to the DVAE used (api.py, `fix_autoregressive_output` function).
*   **Random Voice Quality**: When generating speech with a random voice, the model can be "prone to strange utterances" (tortoise-tts.ipynb).
*   **Limited Sequence Length**: The autoregressive model uses fixed positional encodings, which limits the total amount of speech it can produce in a single pass. The maximum MEL token length is 604 tokens (tortoise/models/autoregressive.py; paper/paper.pdf, p. 12).
*   **Re-ranking on Long Samples**: The CLVP model, used for re-ranking, was trained on clips up to 13 seconds long. As a result, its performance suffers when re-ranking longer audio samples (paper/paper.pdf, p. 12).
*   **Suboptimal Architecture**: The author notes that the diffusion decoder's architecture, which omits feed-forward blocks, was a "poor design decision" and should be improved in future work (paper/paper.pdf, p. 12).
*   **Sampling Rate Discrepancy**: The main model stack operates at 22kHz, but the diffusion decoder outputs MEL spectrograms computed from 24kHz audio. This was done solely for compatibility with the pre-trained UnivNet vocoder (paper/paper.pdf, p. 11).

### Recommendations:
*   **Use a GPU**: A GPU is strongly recommended for reasonable inference times (tortoise-tts.ipynb).
*   **Handle Long Texts**: For generating long-form audio, it is recommended to split the input text into smaller sentences or segments. The repository includes a script with a helper function for this purpose (do_longform_tts.py).
*   **Manage Memory**: If you encounter GPU OOM errors, reduce the `autoregressive_batch_size` (api.py).
*   **Future Improvements**: The author suggests several areas for future work, including using relative positional encodings for arbitrary length sequences, training CLVP on longer audio clips, and improving the diffusion decoder architecture (paper/paper.pdf, p. 12).
