## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by Camb.AI (Source: `hubconf.py`, `LICENSE`).

### Model date:
The model checkpoints are designated as version `v0.1`, but a specific release date is not provided in the repository (Source: `hubconf.py`).

### Model version:
The model version is `v0.1-checkpoints` (Source: `hubconf.py`). The repository describes this as the only pretrained model currently supported (Source: `hubconf.py`).

### Model type:
MARS5 is a Text-to-Speech (TTS) model designed for voice cloning (Source: `inference.py`, `README.md`). It follows a two-stage Autoregressive (AR) and Non-Autoregressive (NAR) architecture (Source: `README.md`).

**Architecture Details:**

*   **Overall Design:** The model takes text and a reference audio as input. An AR model first generates coarse speech features (L0 Encodec codes). These features, along with the text and reference audio, are then fed into an NAR model (a Denoising Diffusion Probabilistic Model - DDPM) to generate the final high-quality speech waveform via a vocoder (Source: `README.md`).
*   **Autoregressive (AR) Component:** The AR model is a Mistral-style decoder-only transformer (`CodecLM`) with a dimension of 1536 (Source: `mars5/model.py`, `README.md`). It is trained to predict the first level (L0) of Encodec quantization codes from text and an implicit speaker embedding derived from the reference audio (Source: `README.md`). The speaker embedding is generated by a small encoder-only transformer (Source: `README.md`, `mars5/model.py`).
*   **Non-Autoregressive (NAR) Component:** The NAR model is an encoder-decoder transformer (`ResidualTransformer`) with a dimension of 1024, trained within a multinomial diffusion framework (Source: `mars5/model.py`, `README.md`). It predicts the remaining 7 Encodec codebooks, conditioned on the AR model's L0 code predictions, the input text, and a speaker embedding (Source: `README.md`). The diffusion process uses a cosine schedule (Source: `mars5/diffuser.py`).
*   **Audio Codec:** The model uses the `EncodecModel` at a 24kHz sampling rate with a target bandwidth of 6.0 kbps (Source: `inference.py`).
*   **Vocoder:** The final audio waveform is generated using a `Vocos` vocoder, specifically `charactr/vocos-encodec-24khz` (Source: `inference.py`).
*   **Tokenizers:** The model employs two types of tokenizers:
    *   A `RegexTokenizer` with a GPT-4 split pattern for processing input text (Source: `inference.py`, `mars5/minbpe/regex.py`).
    *   A `CodebookTokenizer` for processing the speech codec tokens (Source: `inference.py`, `mars5/minbpe/codebook.py`).

### Training details:
*   **AR Model Training:** The autoregressive component is trained using a standard next-token prediction task with a cross-entropy loss. The loss is weighted to give less importance to text tokens compared to acoustic tokens (Source: `README.md`).
*   **NAR Model Training:** The non-autoregressive component is trained using a multinomial diffusion framework. During training, the ground truth L0 codes are used to override the noised input at each diffusion timestep (Source: `README.md`). The diffusion process follows a cosine schedule (Source: `mars5/diffuser.py`).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The primary intended use of MARS5 is to perform Text-to-Speech (TTS) synthesis that emulates or "clones" a speaker's voice from a short reference audio clip (Source: `README.md`, `inference.py`).

**Input-Output Structure:**
*   **Input:** The model's `tts` function requires:
    1.  `text` (str): The text to be synthesized.
    2.  `ref_audio` (Tensor): A 24kHz audio tensor of the reference speaker.
    3.  `ref_transcript` (str, optional): The transcript of the reference audio. This is required for "deep clone" mode.
    4.  `cfg` (InferenceConfig, optional): A configuration object to control synthesis parameters like temperature, top_k, etc. (Source: `inference.py`).
*   **Output:** The function returns:
    1.  `ar_codes` (Tensor): A tensor of discrete coarse code outputs from the AR model.
    2.  `out_wav` (Tensor): The final synthesized audio waveform as a float tensor sampled at 24kHz (Source: `inference.py`).

**Capabilities:**
The model supports two main modes of voice cloning:
1.  **Shallow Clone:** This mode uses the reference audio only to generate an implicit speaker embedding, which conditions the TTS output. It is faster and does not require a transcript of the reference audio (Source: `README.md`).
2.  **Deep Clone:** This mode uses both the reference audio and its transcript. It concatenates the reference transcript with the target text and the reference audio tokens with the input sequence. This provides more accurate cloning of the speaker's identity and prosody but requires a transcript and has higher latency (Source: `README.md`, `inference.py`).

### Primary intended users:
The model is intended for developers, researchers, and practitioners in the field of speech synthesis and audio generation who need to create synthetic speech in a specific voice.

### Out-of-scope uses:
The repository does not explicitly list out-of-scope uses. However, as a voice cloning model, potential misuse includes creating synthetic speech of individuals without their consent, which could be used for malicious purposes such as creating deepfakes, misinformation, or impersonation.

## How to Use
This section outlines how to use the model.

The following code snippet, adapted from the example notebook, demonstrates how to load the model and synthesize audio (Source: `example.ipynb`).

**1. Installation:**
First, install the required dependencies.
```bash
pip install torch torchaudio numpy vocos encodec librosa
```
(Source: `dependencies`, `example.ipynb`)

**2. Load Model and Reference Audio:**
Load the MARS5 model from PyTorch Hub and prepare a reference audio file. The reference audio should be a 24kHz mono waveform.

```python
import torch
import librosa
import IPython.display as ipd

# Load the MARS5 model and its configuration class
mars5, config_class = torch.hub.load('Camb-ai/mars5-tts', 'mars5_english', trust_repo=True)

# Load a reference audio file (e.g., 'example.wav')
# The model expects a 24kHz sample rate, but librosa can resample it.
# The internal sample rate of the model is 24000 Hz.
wav, sr = librosa.load('example.wav', sr=mars5.sr, mono=True)
wav = torch.from_numpy(wav)

# If using deep_clone, provide the transcript for the reference audio
ref_transcript = "We actually haven't managed to meet demand."

print("Reference audio:")
ipd.display(ipd.Audio(wav.numpy(), rate=mars5.sr))
print(f"Reference transcript: {ref_transcript}")
```
(Source: `example.ipynb`, `inference.py`)

**3. Configure and Run TTS Inference:**
Create an `InferenceConfig` object to control the synthesis process. Set `deep_clone=True` for higher fidelity cloning if the reference transcript is available.

```python
# Set deep_clone to True for better quality, or False for faster inference without a transcript.
deep_clone = True

# Configure inference settings.
cfg = config_class(
    deep_clone=deep_clone,
    rep_penalty_window=100,
    top_k=100,
    temperature=0.7,
    freq_penalty=3
)

# Text to synthesize
target_text = "The quick brown rat."

# Perform TTS
ar_codes, wav_out = mars5.tts(
    target_text,
    wav,
    ref_transcript,
    cfg=cfg
)

# Display the synthesized audio
print('Synthesized output audio:')
ipd.display(ipd.Audio(wav_out.numpy(), rate=mars5.sr))
```
(Source: `example.ipynb`, `inference.py`)

**Inference Configuration:**
The `InferenceConfig` class allows tuning of many parameters for both the AR and NAR stages, including:
*   `temperature`, `top_k`, `top_p` for sampling.
*   `freq_penalty`, `presence_penalty` to control repetition.
*   `deep_clone` to switch between cloning modes.
*   `max_prompt_dur` (default 12 seconds) for the maximum recommended reference audio duration.
(Source: `inference.py`)

## Factors
This section addresses variables that may impact the model's performance.

### Relevant factors:
*   **Reference Audio Quality:** The quality of the synthesized speech is dependent on the quality of the reference audio provided for voice cloning.
*   **Reference Audio Duration:** The model issues a warning if the reference audio duration exceeds `max_prompt_dur` (default 12 seconds), as longer prompts may lead to quality degradation (Source: `inference.py`).
*   **Reference Transcript Accuracy:** For `deep_clone` mode, the accuracy of the provided `ref_transcript` is crucial for high-fidelity cloning (Source: `inference.py`, `README.md`).
*   **Cloning Mode:** The choice between `deep_clone` and shallow clone impacts both performance and inference time. Deep cloning improves intelligibility and speaker similarity but requires a transcript and is slower (Source: `inference.py`, `README.md`).

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
*   **Limited Documentation:** The developers note that a full, formal write-up of the model's design and training is not available (Source: `README.md`).
*   **Language Support:** The model is provided as `mars5_english`, indicating it is intended for English language synthesis (Source: `hubconf.py`).
*   **Reference Audio Length:** Using reference audio longer than the recommended maximum (`max_prompt_dur`, default 12s) can degrade output quality (Source: `inference.py`).
*   **Inference Failures:** The model may fail to generate coherent audio for very long inputs, indicated by a warning when the maximum generation length is reached (Source: `ar_generate.py`).
*   **Beam Search:** Only a `beam_width` of 1 is currently supported for autoregressive decoding (Source: `ar_generate.py`).

### Recommendations:
*   **Reference Audio:** For best results, use a high-quality, clean recording of the target speaker that is shorter than 12 seconds (Source: `inference.py`).
*   **Deep Cloning:** For the highest fidelity voice cloning, use the `deep_clone=True` option and provide an accurate transcript of the reference audio (Source: `inference.py`, `README.md`).
*   **Improving NAR Performance:** The developers suggest that users can implement RePaint inpainting techniques by increasing the `jump_len` and `jump_n_sample` parameters in the diffusion configuration to potentially improve the quality of the NAR model's output (Source: `README.md`).
*   **Silence Trimming:** The final output audio is trimmed to remove leading and trailing silences based on a `top_db` threshold, which can be configured in the `InferenceConfig` (Source: `inference.py`, `mars5/trim.py`).
