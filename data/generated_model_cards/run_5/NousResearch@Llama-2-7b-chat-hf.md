## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by Meta (LICENSE.txt). Depending on the user's location, "Meta" refers to Meta Platforms Ireland Limited (for users in the EEA or Switzerland) or Meta Platforms, Inc. (for users outside the EEA or Switzerland) (LICENSE.txt).

### Model date:
The Llama 2 model version was released on July 18, 2023 (LICENSE.txt).

### Model version:
The model is part of the Llama 2 release (LICENSE.txt). The model configuration was generated with `transformers` version "4.31.0.dev0" (config.json, generation_config.json).

### Model type:
The model is a LlamaForCausalLM, which is a foundational large language model of the Llama type, designed for causal language modeling (text generation) (config.json, LICENSE.txt).

**Architecture Details:**
*   **Model Type:** llama (config.json)
*   **Activation Function:** silu (config.json)
*   **Hidden Size:** 4096 (config.json)
*   **Intermediate Size:** 11008 (config.json)
*   **Number of Hidden Layers:** 32 (config.json)
*   **Number of Attention Heads:** 32 (config.json)
*   **Number of Key-Value Heads:** 32 (config.json)
*   **RMS Norm Epsilon:** 1e-05 (config.json)
*   **Initializer Range:** 0.02 (config.json)
*   **Vocabulary Size:** 32000 (config.json)
*   **Maximum Position Embeddings (Context Length):** 4096 tokens (config.json)

**Model Size:**
*   The total size of the model weights is 13,476,835,328 bytes (approx. 13.48 GB) in SafeTensors format (model.safetensors.index.json).
*   The total size of the model weights is 26,953,670,656 bytes (approx. 26.95 GB) in PyTorch format (pytorch_model.bin.index.json).
*   The model uses `float16` precision (config.json).

### Training details:
The model was trained with a pretraining tensor parallelism (`pretraining_tp`) value of 1 (config.json). The default generation parameters are set to `do_sample`: true, `temperature`: 0.9, and `top_p`: 0.6 (generation_config.json). Other specific details about the training algorithm, optimization techniques, or fairness constraints are not available.

### Preprocessing:
The data used for training was preprocessed using a tokenizer with the following steps (tokenizer.json):

*   **Normalization:** Text is normalized by prepending a space and replacing the standard space character (`" "`) with a specific space character (`" "`) (tokenizer.json).
*   **Tokenization Model:** A Byte-Pair Encoding (BPE) model is used for tokenization with a vocabulary of 32,000 tokens. The model includes byte fallback for unknown characters (tokenizer.json).
*   **Special Tokens:** The tokenizer uses special tokens for beginning-of-stream (`<s>`), end-of-stream (`</s>`), and unknown (`<unk>`) (tokenizer.json, special_tokens_map.json).
*   **Post-processing:** A template processor adds a beginning-of-stream (`<s>`) token to the start of single sequences (tokenizer.json).
