## Model Details
This section provides fundamental information about the model, helping stakeholders understand its context and key characteristics.

### Person or organization developing model:
The model was developed by Sanchit Gandhi, Patrick von Platen, and Alexander M. Rush from Hugging Face (2311.00430.pdf, p. 1).

### Model date:
The associated academic paper was submitted to arXiv on November 1, 2023 (2311.00430.pdf, p. 1). The paper details training checkpoints at 10,000 steps (two epochs) and the final models at 80,000 steps (eight epochs) (2311.00430.pdf, p. 11, 6).

### Model version:
This model is `distil-large-v2`. It is a distilled version of the `large-v2` Whisper model (2311.00430.pdf, p. 6, Table 3).

The distilled model is designed to be smaller and faster than the original Whisper model while maintaining performance. `distil-large-v2` has 51% fewer parameters than `large-v2` and is 5.8 times faster on short-form audio transcription (2311.00430.pdf, p. 2, 8). It achieves this by reducing the number of decoder layers from 32 to 2, while keeping the full 32-layer encoder (2311.00430.pdf, p. 6, 12, Table 3, Table 12). This version performs to within 1% Word Error Rate (WER) of the original `large-v2` model on out-of-distribution test data (2311.00430.pdf, p. 1).

### Model type:
Distil-Whisper is a sequence-to-sequence Transformer-based model for Automatic Speech Recognition (ASR) (2311.00430.pdf, p. 1, 3). It is a distilled version of OpenAI's Whisper model (2311.00430.pdf, p. 1).

**Architecture:**
*   **Model:** The model follows the Whisper architecture, which is an encoder-decoder Transformer (2311.00430.pdf, p. 3; config.json.txt).
*   **Encoder:** It uses the full 32-layer encoder from the Whisper `large-v2` model, which is frozen during training. The encoder has a model dimension (`d_model`) of 1280, 20 attention heads, and a feed-forward network dimension of 5120 (2311.00430.pdf, p. 6; config.json.txt).
*   **Decoder:** The decoder is compressed to only 2 layers, initialized from the first and last layers of the teacher model's 32-layer decoder. It has a model dimension of 1280, 20 attention heads, and a feed-forward network dimension of 5120 (2311.00430.pdf, p. 6, Figure 1; config.json.txt).
*   **Size:** The `distil-large-v2` model has 756 million parameters (2311.00430.pdf, p. 8, Table 5).
*   **Context Length:** The model has a fixed receptive field corresponding to 30-second audio inputs (`max_source_positions`: 1500, `n_samples`: 480000, `sampling_rate`: 16000) (2311.00430.pdf, p. 5; config.json.txt; preprocessor_config.json.txt). The decoder's maximum target position length is 448 tokens (config.json.txt).

### Training details:
The model was trained using knowledge distillation from the Whisper `large-v2` model (2311.00430.pdf, p. 1).

*   **Algorithm:** The training objective is a weighted sum of two knowledge distillation terms:
    1.  **Pseudo-Labelling (PL) / Sequence-Level KD:** The model is trained using a cross-entropy loss to predict pseudo-labels generated by the teacher model (`large-v2`) (2311.00430.pdf, p. 4).
    2.  **Kullback-Leibler (KL) Divergence / Word-Level KD:** The full probability distribution of the student model is trained to match the teacher's distribution by minimizing the KL divergence at each token position (2311.00430.pdf, p. 4).
*   **Loss Weights:** The final training objective is `L_KD = 0.8 * L_KL + 1.0 * L_PL` (2311.00430.pdf, p. 4).
*   **Pseudo-Label Filtering:** To ensure high-quality training data, a filtering heuristic was used. Pseudo-labels generated by the teacher model were compared to the ground-truth labels from the original datasets. If the normalized Word Error Rate (WER) between them exceeded a threshold of 10%, the sample was discarded (2311.00430.pdf, p. 4, 11).
*   **Hyperparameters:**
    *   **Batch Size:** 256 (2311.00430.pdf, p. 6, Table 15).
    *   **Optimization Steps:** 80,000 (equivalent to eight epochs) (2311.00430.pdf, p. 6, Table 15).
    *   **Optimizer:** AdamW with β1=0.9 and β2=0.999 (2311.00430.pdf, p. 22, Table 15).
    *   **Learning Rate Schedule:** Slanted triangular learning rate (STLR), with a linear warmup from 0 to 1e-4 over the first 500 steps, followed by a linear decay to 0 (2311.00430.pdf, p. 23).
    *   **Precision:** bfloat16 (2311.00430.pdf, p. 22, Table 15).
    *   **KL Softmax Temperature:** 2.0 (2311.00430.pdf, p. 22, Table 15).
*   **Architecture Initialization:** The 32-layer encoder was copied entirely from the `large-v2` teacher model and frozen during training. The 2-layer student decoder was initialized by copying the weights from the 1st and 32nd layers of the teacher's decoder (2311.00430.pdf, p. 4, 6).

## Intended Use
This section outlines the intended applications of the model.

### Primary intended uses:
The primary intended use of Distil-Whisper is for Automatic Speech Recognition (ASR) in English, particularly in low-latency or resource-constrained environments where running larger models is challenging (2311.00430.pdf, p. 1).

**Capabilities:**
*   **Fast Transcription:** The model is 5.8 times faster than the Whisper `large-v2` model for short-form audio and up to 9.9 times faster for long-form audio using chunking and batching (2311.00430.pdf, p. 2, 5, Table 5).
*   **Robust Performance:** It maintains robust performance on out-of-distribution (OOD) audio, performing to within 1% WER of the original `large-v2` model (2311.00430.pdf, p. 1). It is also robust to noisy acoustic conditions (2311.00430.pdf, p. 2).
*   **Long-Form Transcription:** The model can be used with a chunked transcription algorithm for long audio files and is less prone to hallucination errors than the original Whisper model (2311.00430.pdf, p. 1, 5).
*   **Speculative Decoding:** Distil-Whisper is designed to be used as an "assistant model" for speculative decoding with the full Whisper model. This yields a 2x speed-up while mathematically ensuring the exact same outputs as the original model, making it a drop-in replacement for existing Whisper pipelines (2311.00430.pdf, p. 1, 2).

**Input-Output Structure:**
*   **Input:** The model takes a 30-second audio input, sampled at 16kHz (2311.00430.pdf, p. 5; preprocessor_config.json.txt).
*   **Output:** The model outputs a sequence of text tokens representing the transcription of the audio (2311.00430.pdf, p. 3).

### Primary intended users:
The primary users are developers and researchers in the field of speech recognition who require a fast and efficient ASR model that retains the robustness of larger models (2311.00430.pdf, p. 1).

### Out-of-scope uses:
*   **Non-English Languages:** This model (`distil-large-v2`) is distilled from `Whisper-large-v2` but trained on English pseudo-labels. The training data is English-only, and the model is evaluated on English datasets. Therefore, it is not intended for speech recognition in other languages (2311.00430.pdf, p. 4, 5).
*   **Timestamp Prediction:** The model was trained without predicting timestamps, so it cannot be used for word-level timestamping tasks (2311.00430.pdf, p. 23).
*   **Translation:** The model is trained for transcription (`<|transcribe|>`) and not translation (`<|translate|>`) (tokenizer_summary.json.txt).

*   **Noise Levels:** The model's robustness is evaluated by adding varying levels of white noise and pub noise (measured in SNR) to the input audio (2311.00430.pdf, p. 7, Figure 2).

## Metrics
This section describes how the model's performance is evaluated.

### Model performance measures:
*   **Word Error Rate (WER):** This is the primary metric used to assess transcription accuracy. Text is normalized using the Whisper English normaliser before WER calculation to focus on transcription errors rather than formatting differences (2311.00430.pdf, p. 1, 23).
*   **Relative Error Rate (RER):** Used to measure "effective robustness" by comparing performance on OOD datasets relative to a reference ID dataset (2311.00430.pdf, p. 9, Table 6).
*   **Long-Form Metrics:** For long-form audio, the following metrics are used to quantify hallucinations and other errors (2311.00430.pdf, p. 8, Table 8):
    *   **5-gram word duplicates (5-Dup.):** Measures the number of repetitive sequences.
    *   **Insertion Error Rate (IER):** Quantifies the frequency of hallucinated text.
    *   **Substitution Error Rate (SER):** Quantifies substitution errors.
    *   **Deletion Error Rate (DER):** Quantifies deletion errors.
*   **Latency:** Measured as a relative speed-up compared to the `large-v2` baseline, and as a Real-Time Factor (RTF) (2311.00430.pdf, p. 8, 29, Table 5, Table 24).

### Decision thresholds:
A WER threshold (`λ`) of 10% was used during the data preparation phase. Training samples where the WER between the ground-truth text and the teacher-generated pseudo-label exceeded this threshold were discarded. This was found to provide a good trade-off between data quantity and quality (2311.00430.pdf, p. 4, 11).

### Variation approaches:
Performance is reported as macro-averages across multiple datasets to provide a comprehensive measure of robustness. Specifically, results are averaged over 11 in-distribution (ID) datasets and 4 out-of-distribution (OOD) datasets for short-form evaluation, and across 4 OOD datasets for long-form evaluation (2311.00430.pdf, p. 8, Table 5, Table 16).

## Evaluation Data
This section provides details about the datasets used to evaluate the model.

### Datasets:
The model was evaluated on a total of 15 short-form and 5 long-form datasets.

**Short-Form Out-of-Distribution (OOD) Datasets (2311.00430.pdf, p. 7, Table 4):**
*   **CHIME-4:** 7 hours, 87 speakers, news broadcast domain.
*   **Earnings-22:** 115 hours, unknown speakers, financial meetings domain.
*   **FLEURS:** 2 hours, 3 speakers, narrated Wikipedia domain.
*   **SPGISpeech:** 100 hours, unknown speakers, financial meetings domain.

**Long-Form Out-of-Distribution (OOD) Datasets (2311.00430.pdf, p. 7, Table 4):**
*   **Earnings-21:** 39 hours, unknown speakers, financial meetings domain.
*   **Earnings-22:** 115 hours, unknown speakers, financial meetings domain.
*   **Meanwhile:** 1 hour, 1 speaker, TV show domain.
*   **Rev 16:** 16 hours, unknown speakers, podcast domain.

**In-Distribution (ID) Datasets:**
*   The model was also evaluated on the test splits of the 11 training datasets, including AMI, Call Home, Common Voice 13, GigaSpeech, LibriSpeech, People's Speech, SwitchBoard, TED-LIUM, and Voxpopuli (2311.00430.pdf, p. 7, 24, Table 16).

### Motivation:
The datasets were chosen to cover a broad range of acoustic conditions, speakers, speaking styles, and domains. The use of both in-distribution (ID) and out-of-distribution (OOD) datasets was motivated by the need to measure the model's ability to generalize effectively beyond its training data and retain the robustness of the original Whisper model (2311.00430.pdf, p. 7).

### Preprocessing:
For evaluation, all transcriptions (both predicted and ground-truth) are standardized using the Whisper English normaliser. This process removes or converts specific words, symbols, and numeric expressions, and manages whitespace and spelling to ensure that the WER metric penalizes actual transcription errors rather than formatting differences (2311.00430.pdf, p. 23). For long-form evaluation, audio files are chunked into smaller segments with overlaps (2311.00430.pdf, p. 5).

## Training Data
This section provides details about the datasets used to train the model.

### Datasets:
The model was trained on a large-scale corpus assembled from nine publicly available English speech recognition datasets, totaling 21,770 hours of audio from over 18,260 speakers (2311.00430.pdf, p. 5, Table 2, Table 13). The datasets are:
*   **People's Speech:** 12,000 hours (Government, interviews)
*   **GigaSpeech:** 2,500 hours (Audiobook, podcast, YouTube)
*   **Common Voice 13:** 2,400 hours (Narrated Wikipedia)
*   **Fisher:** 1,960 hours (Telephone conversations)
*   **LibriSpeech:** 960 hours (Audiobooks)
*   **VoxPopuli:** 540 hours (European Parliament)
*   **TED-LIUM:** 450 hours (TED talks)
*   **SwitchBoard:** 260 hours (Telephone conversations)
*   **AMI:** 100 hours (Meetings)

### Motivation:
The datasets were combined to create a large, diverse corpus for large-scale knowledge distillation. This approach was inspired by SpeechStew and aims to ensure the distilled model maintains robustness across different acoustic conditions, domains, and speakers, similar to the original Whisper model which was trained on a massive, diverse web-scraped dataset (2311.00430.pdf, p. 1, 5).

### Preprocessing:
The core of the data preprocessing involved generating high-quality pseudo-labels for distillation (2311.00430.pdf, p. 1).
1.  **Pseudo-Labelling:** All 21,770 hours of training audio were transcribed using the Whisper `large-v2` model with greedy decoding to generate "pseudo-labels" (2311.00430.pdf, p. 5).
2.  **Filtering:** A simple Word Error Rate (WER) heuristic was applied to filter the pseudo-labels. The WER was computed between the normalized ground-truth transcription and the normalized pseudo-label for each sample. Samples with a WER greater than 10% were discarded. This filtering removed 45.4% of the training data, leaving the highest quality pseudo-labels for training (2311.00430.pdf, p. 4, 11, Table 10).
3.  **Audio Processing:** Audio data was loaded and pre-processed using the Hugging Face Datasets library (2311.00430.pdf, p. 5). For the AMI dataset, long audio files were segmented into shorter utterances suitable for training (2311.00430.pdf, p. 21).

## Memory or Hardware Requirements
This section outlines the memory or hardware requirements for loading, deploying, and training the model.

### Loading Requirements:
The `distil-large-v2` model has 756M parameters, which is 51% fewer than the `large-v2` model (1550M parameters) (2311.00430.pdf, p. 2, 8, Table 3, Table 5). When used for speculative decoding with `large-v2`, the total parameter count increases by only 8% because the encoder can be shared between the two models (2311.00430.pdf, p. 10).

### Deploying Requirements:
Inference latency was benchmarked on two types of GPUs with float16 precision:

*   **NVIDIA A100 40GB GPU:**
    *   **Short-form (batch size 1):** 5.8 times faster than `large-v2`. Real-Time Factor (RTF) is 25.1 x 10⁻³ without Flash Attention 2, and 24.2 x 10⁻³ with it (2311.00430.pdf, p. 8, 30, Table 5, Table 24).
    *   **Long-form (batch size 16):** RTF is 6.5 x 10⁻³ without Flash Attention 2, and 4.4 x 10⁻³ with it (2311.00430.pdf, p. 30, Table 24).
*   **NVIDIA T4 16GB GPU:**
    *   **Short-form (batch size 1):** RTF is 38.0 x 10⁻³ without Flash Attention, and 27.4 x 10⁻³ with it (2311.00430.pdf, p. 30, Table 25).
    *   **Long-form (batch size 16):** RTF is 31.2 x 10⁻³ without Flash Attention, and 20.1 x 10⁻³ with it (2311.00430.pdf, p. 30, Table 25).

### Training or Fine-tuning Requirements:
The model was trained using data parallelism across TPU v4-8 accelerators with bfloat16 precision and gradient checkpointing (2311.00430.pdf, p. 22).

## Ethical Considerations
This section discusses the ethical considerations in model development, including challenges, risks, and solutions. 

The provided paper does not contain a dedicated section on ethical considerations. The following points are inferred from the technical content:

*   **Data Usage:** The model was trained on a combination of nine publicly available speech recognition datasets. The licenses for these datasets vary, including CC-BY-SA, CC0, Apache-2.0, and some proprietary LDC licenses (2311.00430.pdf, p. 6, Table 2).
*   **Risk of Hallucination:** The original Whisper model is known to be susceptible to "hallucinations," where it generates text that is not present in the audio, especially during long periods of silence. This is a potential risk in applications requiring high factual accuracy. The development of Distil-Whisper serves as a mitigation strategy, as it was found to be "less prone to hallucination errors on long-form audio" and has a lower insertion error rate (IER) than the teacher model (2311.00430.pdf, p. 1, 8, 11, Table 8).
*   **General ASR Risks:** As with any ASR model, there is a risk of mis-transcription. The impact of such errors depends on the application's context. The paper demonstrates that while the overall WER is close to the state-of-the-art teacher model, the distilled model has a higher substitution error rate (SER), indicating it is more prone to substituting words (2311.00430.pdf, p. 11, Table 8). Users in critical applications should be aware of this trade-off.

## Caveats and Recommendations
This section lists unresolved issues and provides guidance for users.

### Caveats:
*   **Performance Trade-off:** While `distil-large-v2` is significantly faster, its average Word Error Rate (WER) on out-of-distribution (OOD) short-form audio is 1% higher than the `large-v2` teacher model (10.1% vs 9.1%) (2311.00430.pdf, p. 8, Table 5).
*   **Error Types:** The distilled model is more susceptible to substitution errors (SER is 1.4% higher) than the teacher model, even though it makes fewer insertion errors (hallucinations) (2311.00430.pdf, p. 11, Table 8).
*   **Encoder Distillation:** Further reducing the model size by distilling the encoder (e.g., from 32 to 16 layers) leads to a significant degradation in performance (3.1% absolute WER increase), suggesting that a deep encoder is crucial for maintaining strong WER performance (2311.00430.pdf, p. 12, Table 12).
*   **Speculative Decoding with Batching:** The latency benefits of speculative decoding diminish at higher batch sizes. For a batch size of 16, it can perform worse than using the main model by itself (2311.00430.pdf, p. 28, Table 22).
*   **Chunk Length for Long-Form:** The optimal chunk length for long-form transcription differs from the original Whisper model. Distil-Whisper performs best with a 15-second chunk length on the TED-LIUM validation set, whereas Whisper is optimal at 30 seconds (2311.00430.pdf, p. 29, Table 23).

### Recommendations:
*   **Use Case:** Distil-Whisper is recommended as a drop-in replacement for Whisper models in pipelines where lower latency and reduced computational cost are critical, and a minor trade-off in accuracy is acceptable (2311.00430.pdf, p. 1, 8).
*   **Speculative Decoding:** For applications requiring the exact same output as the `large-v2` model but with improved latency, it is recommended to use `distil-large-v2` as an assistant model for speculative decoding. This provides a 2x speed-up at a batch size of 1 (2311.00430.pdf, p. 2, 11).
*   **Flash Attention:** To maximize inference speed, especially at higher batch sizes, it is recommended to use an implementation with Flash Attention 2 (2311.00430.pdf, p. 29).
*   **Long-Form Transcription:** When transcribing long audio files, use the chunked transcription algorithm and consider tuning the `chunk_length_s` hyper-parameter. A shorter chunk length of 15 seconds may be optimal for Distil-Whisper (2311.00430.pdf, p. 28-29).
